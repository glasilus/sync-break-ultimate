## 1. Обзор

В репозитории две программы, и они делают похожую работу:

- **Disc VPC 01** *(офлайн-редактор)* — вы даёте ей трек и одно или несколько исходных видео. Она слушает музыку, сама решает где должны быть склейки и эффекты, и сохраняет готовый видеофайл. Запустили — ушли заваривать чай — пришли, забрали клип.
- **Disc VPC 01 R-T** *(реалтайм, «R-T»)* — то же самое, но вживую. Программа слушает звук, который сейчас идёт у вас (микрофон у колонок, «системный звук» компьютера, выход с микшера) и в реальном времени крутит видео и эффекты на экране. Сделана для VJ-сетов и стримов.

Чтобы пользоваться, не нужно уметь программировать. Обе программы делят одно общее «ухо» — ту часть, которая слушает музыку и решает, что считать битом, нарастанием или дропом — и одну библиотеку визуальных эффектов.

## 2. Установка

> **Просто хотите запустить?** Пропустите этот раздел и скачайте готовый бандл со страницы Releases на GitHub (см. *Готовые сборки* ниже). Сборка из исходников нужна только если вы планируете править код или для вашей платформы нет релиза.

### Офлайн-редактор (Python)

```
Требуется Python 3.10.x. Python 3.11+ может вызвать ошибки зависимостей.
```

```bash
pip install -r requirements.txt
python -m vpc
```

Команды выше работают на всех ОС. Конкретный запуск для каждой:

| ОС | Команда запуска (из корня репо) |
|---|---|
| Windows | `.venv\Scripts\python.exe -m vpc` |
| Linux | `.venv/bin/python -m vpc` |
| macOS | `.venv/bin/python -m vpc` |

В составе `imageio-ffmpeg` идёт встроенный ffmpeg-бинарник для всех поддерживаемых платформ — отдельно ставить ffmpeg в систему не нужно. Все ОС-специфичные нюансы (подавление консольного окна ffmpeg на Windows, поиск ffprobe.exe vs ffprobe, `.venv/Scripts` vs `.venv/bin`) уже обрабатываются внутри кода.

**Доступность HW-кодеров по ОС:** GUI фильтрует список кодеков через `ffmpeg -encoders` при старте, поэтому на каждой платформе видны только кодеры, которые реально поддерживает её сборка ffmpeg.

| ОС | Доступные HW-кодеры |
|---|---|
| Windows | NVENC (NVIDIA), QSV (Intel), AMF (AMD) |
| Linux | NVENC (NVIDIA), QSV (Intel) |
| macOS | VideoToolbox (Apple Silicon / Intel Macs) |

> **AMD на Linux:** AMD на Linux использует VAAPI, для которого нужен hwframe-upload (шаг в графе фильтров). Текущий pipe-sink не настраивает это, поэтому пользователи AMD на Linux откатываются на libx264 — всё ещё получают плюсы от yuv420p pipe, source cache, ETA и Quality-пресетов. Если нужно HW-ускорение AMD на Linux — открывайте issue, добавим VAAPI ingress.

**Ключевые Python-зависимости:**

| Пакет | Назначение |
|---|---|
| `librosa` | Аудиоанализ, детекция атак |
| `opencv-contrib-python` | Ввод/вывод видео и обработка изображений |
| `scipy` / `numpy` | Цифровая обработка сигналов |
| `scenedetect` | Детекция границ сцен |
| `opensimplex` | Шумовые функции для эффектов |
| `Pillow` | Работа с изображениями |
| `imageio-ffmpeg` | FFmpeg-бэкенд для экспорта видео |

### Реалтайм R-T (C++)

R-T — нативное C++ приложение, собираемое через CMake + vcpkg. Оно не делит
runtime с Python-редактором — каждое самодостаточно.

```bash
# 1. Bootstrap vcpkg (один раз, в любой директории на машине)
git clone https://github.com/microsoft/vcpkg.git
./vcpkg/bootstrap-vcpkg.sh         # или .bat на Windows

# 2. Конфигурация и сборка R-T
cmake -S rt_cpp -B rt_cpp/build \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build rt_cpp/build --config Release

# Windows: статический triplet лежит в rt_cpp/triplets/.
# Добавьте  -DVCPKG_TARGET_TRIPLET=x64-windows-static-release
#          -DVCPKG_OVERLAY_TRIPLETS=rt_cpp/triplets
```

## 3. Disc VPC 01 — Офлайн-редактор

**Если кратко:** загрузите песню, загрузите видео (одно или несколько), выберите пресет, нажмите DRAFT для быстрого превью, подкрутите по вкусу, нажмите RENDER FULL VIDEO. Это весь рабочий цикл.

> **Первый запуск долгий.** `DiscVPC01.exe` — самораспаковывающийся бандл: после двойного клика первое окно может появиться через **2–3 минуты**. Ничего не сломалось — Python, встроенный ffmpeg и аудио-библиотеки распаковываются в фоне. Последующие запуски в этой же сессии быстрее, но не мгновенные. На первом старте просто подождите.

### 3.1 Интерфейс

Окно разделено на три вертикальные панели:

| Панель | Содержимое |
|---|---|
| **Левая (сайдбар)** | Загрузка файлов, пресеты, кнопки рендера |
| **Центральная** | Вкладки: EFFECTS / EXPORT / FORMULA / MYSTERY |
| **Правая** | Предпросмотр, прогресс-бар, лог |

В верхней части вкладки **EFFECTS** есть чекбокс **Hide color-altering effects** — см. §3.10b.

### 3.2 Загрузка файлов

Перед рендером необходимо загрузить два файла:

**Load Audio (WAV / MP3)** — аудиодорожка, которая управляет всем анализом. Движок читает файл целиком до начала рендера. Поддерживаемые форматы: `.wav`, `.mp3` и любые форматы, читаемые `librosa` (почти все).

**Load Source Video** — видеоклип, из которого будут нарезаться и обрабатываться кадры. Принимается любое разрешение; разрешение выходного файла задаётся в настройках экспорта.

Зелёная точка (●) рядом с каждой кнопкой подтверждает успешную загрузку файла.

Дополнительно можно загрузить **папку с оверлеями** через вкладку Overlays (см. §3.9).

### 3.3 Аудиоанализ

Прежде чем что-то резать, программа слушает вашу песню от начала до конца и составляет своего рода карту: где удары барабанов, где сидит бит, какие места громкие, какие тихие, какие шумные, а какие чистые. Дальше склейки и эффекты опираются на эту карту. Делать вручную ничего не нужно — анализ запускается сам при первом нажатии любой кнопки рендера.

Под капотом используется `librosa` и проходят следующие шаги:

1. **Детекция атак (Onset detection)** — выявляет транзиентные удары в сигнале (удары барабанов, атаки нот и т.д.)
2. **Отслеживание бита (Beat tracking)** — извлекает сетку битов для привязки к ритму
3. **Вычисление RMS** — рассчитывает среднеквадратичную энергию для каждого сегмента
4. **Спектральная плоскостность** — измеряет, насколько спектр «похож на шум» (белый шум = 1.0, чистый тон ≈ 0)
5. **Классификация сегментов** — присваивает каждому сегменту один из шести типов (см. §5)

Анализ выполняется один раз за сессию для каждого аудиофайла. Результаты кешируются в памяти для повторных рендеров.

### 3.4 Логика монтажа

Секция **Cut Logic** (первая аккордеонная группа во вкладке EFFECTS) управляет тем, как движок выбирает и выстраивает видеоклипы — то есть *когда* резать и *в какой* кадр.

---

**Smart Scene Detection** *(вкл/выкл)*

При включении движок использует `PySceneDetect` для поиска границ сцен в исходном видео до начала рендера. Клипы выбираются с точек перехода, а не случайных кадров — это даёт более визуально связные переходы.

Рекомендуется для материала с отчётливыми сценами (кино, съёмочный материал). Отключите для абстрактного или повторяющегося видео.

---

**Global Chaos Level** *(0.0 – 1.0)*

Управляет общей случайностью выбора клипов. При `0.0` клипы выбираются предсказуемо на основе детекции сцен. При `1.0` каждая склейка выбирает случайный кадр независимо от сцен. Также масштабирует вероятность срабатывания большинства эффектов.

---

**Beat Threshold** *(0.5 – 2.0)*

Множитель, применяемый к среднему RMS для определения того, является ли сегмент «громким». Сегмент считается громким, если:

```
rms_сегмента > rms_среднее × threshold
```

Меньшие значения → больше сегментов считаются громкими → больше склеек и эффектов. Большие значения → только самые пиковые моменты вызывают реакцию.

**Рекомендуемые начальные значения:**
- Тихий или динамичный материал: `0.9 – 1.1`
- Плотный/компрессированный материал: `1.3 – 1.6`

---

**Transient Sensitivity** *(0.1 – 1.5)*

Порог для детекции резкой атаки (быстрый рост RMS). Используется для классификации сегментов IMPACT. Меньшие значения детектируют больше транзиентов; большие реагируют только на очень резкие удары.

---

**Min Cut Duration (sec)** *(0.0 – 0.3)*

Сегменты короче этого значения объединяются со следующим и не порождают склейку. Предотвращает микросклейки и мерцание на плотном аудио. Типичный рабочий диапазон: `0.04 – 0.1`.

---

**Scene Buffer Size** *(2 – 30)*

Количество уникальных временных меток сцен, хранящихся в памяти для случайного выбора в хаотичные моменты. Большой буфер даёт больше разнообразия; маленький заставляет монтаж циклически использовать меньше сцен.

---

**Snap Cuts to Beat Grid** *(вкл/выкл)*

При включении каждый обнаруженный атак привязывается к ближайшей позиции на сетке битов, если попадает в окно **Beat Snap Tolerance**. Это делает склейки более точными по ритму.

**Beat Snap Tolerance** *(0.01 – 0.15 сек)* — максимальное расстояние, на которое атак может быть сдвинут к ближайшему биту. Если атак дальше — остаётся на месте. Когда snap включён, обнаруженный глобальный BPM появляется в логе рядом со счётчиком сегментов (`Audio: 198.4s | Segments: 412 | 142.0 BPM`).

---

**Manual BPM** *(чекбокс + числовое поле, 20 – 400)*

Когда автоопределённый BPM ошибочен (типично для треков с редкими барабанами, half-time жанров или когда librosa цепляется за под-гармонику), включите **Use Manual BPM** и введите нужное значение. Движок обходит beat tracking и выкладывает равномерную сетку с заданным темпом — детекция атак по-прежнему работает, но snap-to-beat теперь использует вашу сетку, а не определённую автоматически.

Удобно, когда вы знаете темп из DAW, релиза или любого tap-tempo инструмента. Если чекбокс снят — используется автоопределение.

### 3.4a Анатомия блока эффекта

Каждый эффект во вкладке EFFECTS устроен по одному шаблону. Поняв его один раз, вы знаете, как пользоваться всеми 40+ эффектами.

```
┌──────────────────────────────────────────────────────────────┐
│ ☑ Название эффекта  [?]                          ☐ always    │
│   Chance                              [════●═══════════] 0.45 │
│   Always-on intensity (видна только когда «always» включён)  │
│                                       [════════●═════] 0.70   │
│   ─ параметры конкретного эффекта ниже ─                     │
│   Param 1                             [══●══════════] 0.30   │
│   Param 2 (combo)                     [Choice ▾]             │
└──────────────────────────────────────────────────────────────┘
```

**Чекбокс enable** — включает эффект. При выключении эффект не работает независимо от любых других настроек.

**Иконка [?]** — наведите курсор, чтобы прочитать описание эффекта во всплывающей подсказке. Подсказки доступны на английском и русском (через разделитель). Почти у каждой надписи, слайдера и чекбокса в программе она есть — не уверены в чём-то, наводите.

**Chance** *(0.0 – 1.0)* — вероятность срабатывания эффекта на подходящем аудиосегменте. `0.0` = никогда, `1.0` = на каждом подходящем сегменте. Каждый эффект декларирует список типов сегментов, на которые реагирует (см. §5) — даже при chance `1.0` он не сработает на сегменте, тип которого не в списке. Отображается заданное пользователем значение, но внутренне оно ещё умножается на **Global Chaos** (главный регулятор), поэтому одинаковый chance даёт более агрессивный результат на высоком хаосе.

Эффекты без слайдера chance (Stutter, Feedback Loop) при включении срабатывают на каждом подходящем сегменте — они спроектированы как постоянная подложка, не точечный удар.

**always** *(чекбокс)* — обходит триггеры по типу сегмента и слайдер chance именно для этого эффекта. Когда включён — эффект работает на **каждом кадре** независимо от аудио. Используйте, когда нужна постоянная базовая обработка (например, всегда сканлайны или фишай), на которую сверху ложатся аудио-реактивные эффекты.

Это персональная настройка эффекта: включение `always` для *Scanlines* не влияет на аудио-реактивность других эффектов.

**Always-on intensity** *(0.0 – 1.0)* — фиксированная интенсивность для режима `always`. Слайдер **скрыт**, когда `always` выключен (нет смысла показывать ручку, которая ни на что не влияет). При выключенном `always` эффект использует интенсивность, выводимую из аудио.

**Параметры конкретного эффекта** — слайдеры и комбобоксы под стандартными контролами. Их смысл описан в посекционных описаниях ниже.

---

### 3.5 Эффекты — Основные

Это самые ходовые визуальные обработки — вспышки, заикания, шлейфы, pixel-sort, datamosh, ASCII. У каждого — переключатель вкл/выкл и, где это уместно, слайдер **chance** — это просто «как часто срабатывать когда повод подходящий». `0.0` = никогда. `1.0` = каждый раз.

Не обязательно понимать внутренности каждого эффекта — включите, нажмите DRAFT, посмотрите. Описания ниже — для случая, когда вам нужен конкретный вид и хочется знать какую крутилку трогать.

---

**Stutter / Drill**

*Триггер: только сегменты IMPACT*

Повторяет первый микросегмент удара 2, 4 или 8 раз подряд, создавая эффект дрилла или заикания. Слайдера chance нет — эффект срабатывает на каждом IMPACT при включении.

---

**Flash Frame**

*Триггер: DROP, IMPACT*

Вставляет короткий белый или чёрный кадр (или смешение к белому/чёрному) в моменты резкого изменения энергии. Интенсивность вспышки масштабируется с intensity-значением сегмента.

**Chance** *(0.0 – 1.0)* — вероятность срабатывания на каждом подходящем сегменте.

---

**Ghost Trails**

*Триггер: SUSTAIN, BUILD*

Смешивает текущий кадр с предыдущим, создавая шлейф движения или эффект призрака. Подходит для медленных нарастаний и длинных сустейнов.

**Opacity** *(0.0 – 1.0)* — насколько сильно предыдущий кадр просвечивает. При `0.0` эффект невидим; при `1.0` изображение практически заморожено.

---

**Pixel Sort**

*Триггер: NOISE, IMPACT*

Сортирует столбцы пикселей в горизонтальных полосах по яркости, тону или насыщенности. Даёт характерную эстетику, напоминающую сбой кодека.

**Chance** *(0.0 – 1.0)* — вероятность на каждый подходящий сегмент.

**Pixel Sort Intensity** *(0.0 – 1.0)* — высота сортируемых полос. При низких значениях затрагиваются только тонкие полосы; при высоких — почти весь кадр.

**Sort Axis** — свойство пикселей, используемое как ключ сортировки:
- `luminance` — сортировка по воспринимаемой яркости (по умолчанию, наиболее читаемо)
- `hue` — сортировка по тону цвета в пространстве HSV (более красочный результат)
- `saturation` — сортировка по чистоте цвета

---

**Datamosh**

*Триггер: NOISE*

В режимах **Draft** и **Preview**: симулирует эффект растекания P-кадров с помощью оптического потока — векторы движения предыдущего кадра применяются для деформации изображения.

В режиме **Final render**: применяется реальное удаление I-кадров на уровне кодека, производя подлинный datamosh-эффект.

**Chance** *(0.0 – 1.0)* — вероятность на каждый подходящий сегмент.

---

**ASCII Filter**

*Триггер: SUSTAIN*

Конвертирует кадр в ASCII-арт, отрисованный обратно как изображение. Каждый символ представляет блок пикселей, плотность символа соответствует яркости блока.

**Char Size (px)** *(4 – 40)* — размер блока символа в пикселях. Меньшие значения дают более детальный результат; большие — более блочный, абстрактный вид.

**Blend (0 – 1)** — накладывает ASCII на исходный кадр. При `0.0` — чистый ASCII. При `1.0` — исходный кадр без изменений. Промежуточные значения смешивают оба.

**ASCII Color Mode:**
- `fixed` — все символы рисуются в заданном цвете переднего плана на заданном фоне
- `original` — каждый символ берёт средний цвет исходных пикселей, которые он представляет
- `inverted` — то же самое, но с инвертированными цветами

**ASCII FG/BG Color** — RGB-слайдеры (0–255) для переднего плана и фона в режиме `fixed`.

---

### 3.6 Эффекты — Глитч

Эти эффекты симулируют деградацию сигнала, артефакты кодека и ошибки передачи.

| Эффект | Триггер | Описание |
|---|---|---|
| **RGB Shift** | IMPACT, BUILD | Смещает красный канал вправо, синий влево на смещение, пропорциональное интенсивности. Хроматическая аберрация. |
| **Block Glitch** | IMPACT, DROP | Заменяет случайные прямоугольные блоки мусорными пикселями или копирует их с случайной позиции. |
| **Pixel Drift** | NOISE, IMPACT | Смещает целые строки пикселей вбок на случайные расстояния, имитируя потерю горизонтальной синхронизации. |
| **Color Bleed / VHS Smear** | NOISE | Размазывает цветовые каналы горизонтально, имитируя аналоговую задержку хромы. |
| **Freeze + Corrupt** | DROP | Замораживает кадр и применяет случайную порчу пикселей, как будто видеопоток завис. |
| **Negative** | IMPACT | Инвертирует все значения цветов (`255 - значение`) на один кадр. |

У каждого эффекта есть слайдер **Chance** — вероятность срабатывания на подходящем сегменте.

---

### 3.7 Эффекты — Сложные

Эти эффекты используют состояние нескольких кадров или более вычислительно затратные трансформации.

---

**Feedback Loop**

*Триггер: SUSTAIN, BUILD — срабатывает на каждом подходящем кадре при включении, без chance*

Смешивает текущий кадр с затухающей копией, накопленной из предыдущих кадров — эффект видеофидбэка. Накопление усиливает сильные сигналы и рассеивает тихие.

---

**Phase Shift** *(Chance)*

Делит кадр на горизонтальные полосы и смещает чередующиеся полосы влево и вправо на нарастающие смещения.

---

**Mosaic Pulse** *(Chance)*

Пикселизирует кадр путём уменьшения и увеличения масштаба. Размер блока управляется значением RMS баса — пикселизация пульсирует с низкочастотной энергией.

---

**Echo Compound** *(Chance)*

Накладывает несколько копий кадра на себя со смещёнными цветовыми оттенками и лёгким пространственным смещением — многослойное хроматическое эхо.

---

**Kali Mirror** *(Chance)*

Применяет калейдоскопическое зеркальное преобразование, отражая и поворачивая квадранты кадра для создания симметричных геометрических узоров.

---

**Glitch Cascade** *(Chance)*

Объединяет несколько глитч-эффектов в цепочку на одном кадре, суммируя их результаты. Конкретная цепочка рандомизируется для каждого кадра.

---

### 3.8 Эффекты — Деградация

Эффекты деградации симулируют изношенные или низкокачественные носители записи/воспроизведения.

| Эффект | Триггер | Описание |
|---|---|---|
| **Scan Lines** | SUSTAIN, NOISE | Рисует тёмные горизонтальные линии через строку, имитируя CRT-монитор. |
| **Bitcrush / Posterize** | Любой | Снижает глубину цвета, квантуя значения пикселей до меньшего числа уровней. |
| **JPEG Crush** | IMPACT, NOISE | Перекодирует кадр с очень низким качеством JPEG и декодирует обратно — блочные артефакты компрессии. |
| **Fisheye / Barrel** | BUILD | Применяет бочкообразное искажение линзы. |
| **VHS Tracking** | NOISE, DROP | Симулирует ошибку трекинга VHS-кассеты: горизонтальное смещение верху и внизу + шум яркости. |
| **Interlace** | SUSTAIN | Разделяет чётные и нечётные строки развёртки, смещая их на несколько пикселей. |
| **Bad Signal** | DROP, NOISE | Накладывает артефакты цифрового распада сигнала — горизонтальные полосы, цветовые сдвиги, шумовые полосы. |
| **Dithering** | SILENCE, SUSTAIN | Применяет упорядоченный паттерн дизеринга, сокращая эффективную цветовую палитру. |
| **Zoom Glitch** | IMPACT | Анизотропное растяжение — выбирается одна ось (X *или* Y) случайно, на кадре-триггере она резко доходит до пиковой шкалы, потом по кубической кривой плавно возвращается в норму за несколько кадров. Это уже не однокадровый импульс, а направленный рывок с плавным возвратом. |
| **Sharpen** | Любой | Unsharp mask. Жёстко поднимает контраст границ. Хорошо подходит для зернистого/мягкого материала, когда хочется ощущения «вырезано из стекла», а не из бархата. |

У каждого эффекта есть слайдер **Chance**.

У **Zoom Glitch** есть дополнительный параметр — **Snap-back Duration (frames)** *(2 – 24)* — определяет, за сколько кадров идёт плавный возврат. Короткие значения ощущаются как удар, длинные — как вдох.

У **Sharpen** два дополнительных: **Amount** *(0 – 3)* — сила unsharp-mask ядра — и **Radius** *(1 – 9)* — радиус гауссового размытия, которое вычитается. Больший радиус = ореолы вокруг более крупных объектов.

---

### 3.9 Эффекты — Оверлеи и Chroma Key

Оверлеи накладывают внешние файлы изображений на видеокадры. Папку с оверлеями загружают через кнопку в сайдбаре или через вкладку Overlays.

**Enable Overlays** *(вкл/выкл + Chance)*

Движок выбирает случайное изображение из папки на каждом подходящем кадре и накладывает его на видео.

**Opacity** *(0.0 – 1.0)* — прозрачность изображения оверлея.

**Scale Max** *(0.05 – 1.0)* — максимальный размер оверлея как доля от размеров кадра.

**Scale Min** *(0.05 – 1.0)* — минимальный размер. Движок выбирает случайный масштаб между Min и Max для каждого появления.

**Blend Mode:**
- `screen` — осветляет изображение (хорошо для бликов, засветок, текстур)
- `normal` — стандартный альфа-композитинг поверх кадра
- `multiply` — затемняет изображение (хорошо для гранжа, зерна плёнки)

**Position:**
- `random` — оверлей размещается в случайной позиции в пределах кадра
- `center` — оверлей центрируется по кадру
- `corners` — оверлей циклически обходит угловые позиции

Поддерживаемые форматы: `.png`, `.jpg`, `.gif` (только первый кадр), `.bmp`, `.mp4`, `.avi`, `.mov`, `.mkv`, `.flv`, `.wmv`, `.mpg`, `.mpeg`

Видеофайлы загружаются покадрово; каждый кадр становится отдельным изображением оверлея в пуле.

---

### Chroma Key (Greenscreen / Хромакей)

Удаляет цвет фона из каждого изображения оверлея перед наложением, делая этот цвет прозрачным.

**Режим (Mode):**
- `none` — хромакей отключён (по умолчанию)
- `dominant` — автоматически определяет и удаляет наиболее часто встречающийся цвет в оверлее (например, зелёный экран)
- `secondary` — удаляет второй по частоте цвет
- `manual` — удаляет цвет, заданный RGB-слайдерами ниже

**Tolerance** *(5 – 60)* — ширина диапазона оттенков вокруг ключевого цвета, который вырезается. Увеличьте, если края зелёного экрана удаляются не полностью; уменьшите, чтобы не захватывать лишнее.

**Edge Softness** *(1 – 21)* — применяет гауссово размытие к краю маски ключа, сглаживая переход между вырезанными и невырезанными областями.

**Manual Key Color (R / G / B)** *(0 – 255 каждый)* — целевой цвет для режима `manual`. По умолчанию — чистый зелёный (0, 255, 0).

---

### 3.10 Эффекты — Сигнальный домен

Группа **SIGNAL DOMAIN** — это экспериментальная полка. Здесь живут более странные, более абстрактные эффекты. Они берут идеи из обработки звука, сжатия файлов и даже клеточных автоматов и применяют их к пикселям так, как эти процессы применять не предполагалось. Результаты — от тонких до полностью разрушающих изображение. Если хочется понять *почему* каждый эффект выглядит так, описания ниже объясняют технику; если просто нужен новый вид — включите и пробуйте.

Все эффекты этой группы срабатывают по типу аудиосегмента и интенсивности — так же, как все остальные эффекты. У каждого есть чекбокс включения, слайдер Chance и один или два слайдера параметров.

---

**Resonant Rows**

Применяет IIR-полосовой фильтр вдоль каждой строки пикселей, трактуя строку как одномерный аудиосигнал. Фильтр создаёт пространственный звон на вертикальных границах — не размытие, не повышение резкости, а резонанс: узкочастотное эхо вокруг каждой границы. При высоких значениях Q звон поглощает исходное изображение.

- *Freq* — центральная частота резонанса (циклов на пиксель). Ниже = резонируют более широкие горизонтальные объекты. Выше = резонирует более мелкая деталь.
- *Q factor* — острота пика резонанса. Малый Q = мягкий ореол. Высокий Q = агрессивная осциллирующая бахрома.

---

**Temporal RGB Shift**

Три цветовых канала считываются из разных моментов времени. Красный — всегда текущий кадр. Зелёный — N/2 кадров назад. Синий — N кадров назад. Движущиеся объекты отбрасывают цветоразделённые тени через собственную историю. Статичные изображения не затрагиваются.

Это не то же самое, что RGB Shift (который пространственный). Это хроматическая временна́я аберрация.

- *Lag (frames)* — максимальное временно́е смещение для синего канала. Масштабируется пропорционально аудиоинтенсивности.

---

**FFT Phase Corrupt**

Вычисляет двумерное дискретное преобразование Фурье кадра, добавляет случайный шум к фазе каждой частотной компоненты, оставляя амплитуду нетронутой, затем выполняет обратное преобразование. Изображение теряет пространственную связность, сохраняя общее распределение яркости — границы растворяются в волновых интерференционных узорах.

- *Phase Noise Amount* — величина рандомизации фазы (0 = нет эффекта, 1 = полностью случайная фаза = максимальное разрушение).

---

**Waveshaper / Tube Sat**

Применяет передаточную функцию tanh мягкого ограничения к значениям пикселей — идентично насыщению лампового усилителя в аудиопроизводстве. Средние значения сжимаются к центру; пики насыщаются. Результат — характерная аналоговая теплота/искажение в зависимости от уровня драйва.

- *Drive* — усиление перед стадией ограничения. Малый = мягкое насыщение. Высокий = жёсткое, окрашенное искажение.

---

**Histogram Lag**

Совмещает цветовую гистограмму текущего кадра с гистограммой кадра N кадров назад, используя поканальное ремаппирование кумулятивной функции распределения. Изображение сохраняет структуру, но заимствует цветовую палитру из прошлого. При смене сцен текущий кадр принимает цвета совершенно другого момента.

- *Palette Lag (frames)* — насколько далеко в прошлое уходит эталонная палитра.

---

**Wrong Chroma Sub (4:1:N)**

Конвертирует кадр в цветовое пространство YCrCb, делает даунсэмплинг каналов Cr и Cb с большим коэффициентом (от 2× до 8×), затем возвращает их обратно интерполяцией ближайшего соседа. Результат: крупные блочные цветные пятна поверх чёткой яркостной детали. Это визуальная сигнатура экстремального хромасаблинга — то, что происходит, когда плохой кодек отбрасывает слишком много информации о цвете.

- *Downsample Factor* — агрессивность сокращения хромы. При 2 эффект мягкий; при 8 цвета появляются в крупных прямоугольных блоках.

---

**Game of Life Mask**

Бинаризует кадр в половинном разрешении по порогу яркости, запускает «Жизнь» Конвея на N итераций, используя содержимое изображения как начальное состояние клеток, затем использует живые клетки как маску повреждения — XOR-суммируя шум в эти области.

Более яркие области изображения, как правило, порождают активные клеточные популяции. После нескольких итераций паттерн развивается в глайдеры, натюрморты и осцилляторы Конвея — создавая органичную и полупредсказуемую структуру повреждения, привязанную к содержимому изображения.

- *Iterations* — сколько поколений «Жизни» запускать на каждый кадр. Больше итераций = более эволюционировавший, менее производный от изображения паттерн.

---

**ELA (Error Level Analysis)**

Перекодирует текущий кадр в JPEG с фиксированным качеством, затем вычисляет абсолютную разницу между оригиналом и перекодированной версией с усилением. Области, уже сильно сжатые JPEG, выглядят тёмными. Области с мелкой детализацией, чёткими границами или недавно отредактированным содержимым выглядят яркими. Применённая как слой наложения, это даёт криминалистическую карту границ, раскрывающую пространственную структуру сжатия изображения.

- *ELA Blend* — 0 = чистая карта ELA (исходное изображение заменено картой ошибок), 1 = только оригинал. Промежуточные значения смешивают оба.

---

**Dtype Reinterpret**

Берёт необработанные байты буфера кадра и переинтерпретирует первую половину как 16-битные значения с плавающей точкой. В домене float16 добавляется малый шум. Байты затем переинтерпретируются обратно как беззнаковые 8-битные целые и преобразуются в форму кадра.

Повреждение не случайно — оно определяется конкретными числовыми паттернами в данных пикселей. Определённые комбинации значений пикселей при интерпретации как float16 дают NaN или Inf, которые разрешаются в конкретные байтовые паттерны (0 или 255 в uint8), создавая структурированные светлые и тёмные повреждения, полностью зависящие от содержимого.

- *Noise Amount* — стандартное отклонение шума в домене float16.

---

**Spatial Reverb**

Строит затухающий импульсный отклик (серия убывающих эхо с фиксированными пространственными интервалами) и свёртывает его с каждой строкой пикселей. Результат — горизонтальная реверберация: яркие пиксели порождают затухающие копии себя справа. Это комнатная акустическая реверберация, применённая к пространственному свету.

- *Reverb Decay* — скорость затухания эхо. Высокие значения дают более длинные и громкие реверберационные хвосты.

---

### 3.10b Скрыть эффекты, меняющие цвет

Сверху вкладки **EFFECTS** есть чекбокс **Hide color-altering effects**. При включении он делает три вещи одновременно:

1. Принудительно выключает указанные эффекты (их чекбоксы сбрасываются).
2. Полностью скрывает их блоки в аккордеоне, чтобы их нельзя было случайно включить, пока галочка стоит.
3. Если в *Silence Treatment* был выбран `Dim`, переключает его на `None` и прячет радиокнопку `Dim`.

При снятии галочки предыдущее состояние всех затронутых эффектов — включая `Dim`, если он был активен — восстанавливается из снапшота, сделанного в момент включения. Можно переключаться туда-обратно без потери настроек.

Список эффектов, считающихся «цветовыми» (их и прячет тоггл):

`Flash Frame`, `RGB Shift`, `Color Bleed / VHS Smear`, `Negative`, `Bitcrush / Posterize`, `Glitch Cascade`, `Temporal RGB Shift`, `FFT Phase Corrupt`, `Waveshaper / Tube Sat`, `Dtype Reinterpret`, `ELA`, `Spatial Reverb`.

Используйте, когда оригинальный цветокор исходного видео важен (материал от конкретного оператора, готовый цветокорректированный клип, архив) и нужно положить сверху склейки/глитчи без того, чтобы движок ездил по палитре. По умолчанию **выключено**.

### 3.10c Эффекты — Warp

Группа **WARP** искривляет пиксели через карты смещения. Если эффекты Glitch и Degradation работают локально (попиксельно или построчно), Warp двигает целые регионы изображения — это ближе к оптическому потоку, чем к коррапту. На малой интенсивности — едва заметно, на большой — полностью переписывает изображение.

Все четыре эффекта группы поддерживают режим `always` для постоянного базового искривления.

---

**Deriv Warp (gradient flow)**

*Триггеры: IMPACT, NOISE, DROP, SUSTAIN*

Ближайший CPU-аналог настоящего датамоша. Пространственный градиент предыдущего кадра используется как поле смещения для текущего — движение в источнике порождает «смаз» в том же направлении. С включённым **Feedback Loop** смаз накапливается и даёт настоящий датамош-эффект «волочения».

- *Prev Blend* *(0.0 – 0.6)* — насколько просвечивает предыдущий кадр. `0` — чистое смещение, `0.6` — устойчивый «смаз» с памятью.

---

**Vortex Warp (spiral)**

*Триггеры: BUILD, IMPACT, SUSTAIN, DROP*

Вращает пиксели вокруг центра холста. Угол вращения = `интенсивность × гауссово_затухание(радиус)`, поэтому центр закручивается сильнее, чем края. На малой интенсивности — лёгкий завиток, на большой — изображение коллапсирует в спираль.

Параметров нет — сила определяется аудио (или фиксируется через ползунок always-on intensity).

---

**Fractal Warp (organic)**

*Триггеры: любые типы сегментов*

Строит многооктавное Simplex-шумовое поле и использует его как XY-смещение. Поле перерождается на каждом аудио-сегменте, поэтому форма искажения постоянно меняется, а не залипает в одном паттерне. Даёт органическое «плавление» изображения.

- *Octaves* *(2 – 5)* — количество суммируемых октав шума. `2` — плавные «капли», `5` — рваный фрактал.

---

**Self Displace (auto-warp)**

*Триггеры: IMPACT, NOISE, DROP, BUILD, SUSTAIN*

Канал R прошлого кадра становится X-смещением, канал G — Y-смещением. Изображение буквально разрывает само себя своими же цветами. В паре с `Feedback` каскадирует в датамош-grade смаз, который реагирует на структуру сцены, а не на случайный шум.

- *Depth (frames back)* *(1 – 8)* — насколько глубоко в прошлом берётся источник смещения. Больше — заметнее «разрыв» от лага.

---

### 3.10d Вкладка FORMULA

Вкладка **FORMULA** (третья в центральной панели, палитра BSOD-blue) — редактор свободных NumPy-выражений для эффекта-формулы. Введите выражение, нажмите `RENDER FULL VIDEO` — движок будет вычислять его на каждом кадре.

Это один эффект (он встаёт в цепочку рядом с остальными), но отдельная вкладка даёт ему больше места, чем обычный блок аккордеона: статус-строка, сниппеты, четыре live-параметра.

**Чекбокс enable + Chance + Blend** — верхняя строка.
- `enable` включает эффект.
- `chance` (0–1) — вероятность срабатывания на подходящем кадре. Эффект-формула слушает все типы сегментов, поэтому chance — единственный гейт сверху Global Chaos.
- `blend` (0–1) — смешивает результат формулы с оригинальным кадром. `0.0` — чистая формула, `1.0` — виден только оригинал (эффект невидим).

**Editor** — многострочное текстовое поле. Введите любое NumPy-выражение, возвращающее массив `H×W×3 uint8`. Выражение перекомпилируется на каждом нажатии клавиши; статус-строка снизу пишет `OK compiled clean — formula ready` при успехе или текст синтаксической ошибки. Ошибка компиляции или рантайма тихо возвращает оригинальный кадр — опечатка не падает рендер, но и эффект не применяется.

**Live params a / b / c / d** — четыре слайдера (0.0–1.0), которые можно использовать в выражении. Удобно, когда хочется «играть» формулой во время рендера — одна и та же формула при `(a=0.1, b=0.7)` и `(a=0.9, b=0.1)` даст совершенно разные картины.

**Snippets** — кликабельные стартовые точки. Клик заменяет содержимое редактора. Доступные:

| Сниппет | Что делает |
|---|---|
| Identity | `frame` — пасс-через; полезно как база для blend |
| Invert | `255 - frame` — цветовой негатив |
| Pulse | яркость пульсирует во времени с `t` |
| Channel sweep | R-канал катится вправо, B — влево, амплитуда от `a · sin(t·3)` |
| Posterize | квантизация цветовой глубины, глубина от `a` |
| Scanlines | каждая вторая строка затемнена на `1-a` |
| Wave | синусоидальное вертикальное смещение |
| Plasma | классическая демосценовая плазма поверх кадра |
| Threshold | бинарный порог, уровень которого осциллирует с `t` |
| Mirror | левая половина зеркалит правую |

**Доступные переменные в пространстве имён:**

| Имя | Тип | Смысл |
|---|---|---|
| `frame` | `(H, W, 3) uint8` | текущий входной кадр |
| `r`, `g`, `b` | `(H, W) uint8` | поканальные виды `frame` |
| `x`, `y` | `(H, W) float32` | сетки координат пикселей |
| `t` | `float` | время начала сегмента в секундах |
| `i` | `float, 0..1` | текущая масштабированная интенсивность |
| `a`, `b`, `c`, `d` | `float, 0..1` | значения live-слайдеров с этой вкладки |
| `np`, `cv2` | модули | полностью доступны NumPy и OpenCV |
| `sin cos tan abs clip sqrt exp log pi` | алиасы | удобные сокращения |

Результат автоматически бродкастится к форме кадра и клиппится в uint8. Возврат одного числа, 1D-массива, `H×W` или `H×W×3` — всё работает.

**Примеры из справки:**

```python
255 - frame                                      # полный инверт
np.roll(frame, int(20 * np.sin(t*5)), 1)        # горизонтальный сдвиг
cv2.GaussianBlur(frame, (15, 15), 0)             # размытие через cv2
frame * (1 + 0.5 * a * np.sin(t*3))              # пульсация по `a`
np.where((y % 2 == 0)[:,:,None], frame, frame//2)  # интерлейс
```

Сохранение формул: само выражение и значения `a/b/c/d` входят в cfg, поэтому при сохранении пресета сохраняются и они. Чтобы поделиться формулой — сохраните её как пресет и передайте JSON-запись.

### 3.11 Настройки экспорта

Расположены во вкладке **EXPORT**.

**Frame Rate (FPS)** — `12 / 24 / 30 / 60`, по умолчанию `24`. Не влияет на тайминг исходного аудио; контролирует только количество генерируемых видеокадров в секунду.

Рядом с выпадающим списком FPS — кнопка **Match Source FPS**. По нажатию программа считывает первое загруженное видео и подставляет его нативную частоту кадров (округлённую до целого) в поле FPS. Полезно и в обычном режиме, и в Passthrough (§3.17) — в Passthrough это и есть способ убедиться, что выход идёт точно в темпе исходника, без передискретизации.

**Resolution Mode** — три политики выбора выходных размеров:

| Режим | Поведение |
|---|---|
| **Preset** | Выбор из `240p / 360p / 480p / 720p / 1080p`. По умолчанию `720p`. |
| **Match source** | Размеры выхода совпадают с первым загруженным исходным видео пиксель в пиксель. |
| **Custom** | Свои ширина и высота в спинбоксах (`64 – 7680` × `64 – 4320`). |

Во всех трёх режимах размеры тихо округляются вниз до ближайшего **чётного** числа — `yuv420p` (и ProRes 422) требуют четырёхугольной субдискретизации цветности по чётным осям, иначе ffmpeg падает на старте. То есть `1281 × 721 → 1280 × 720`. Это касается и `Match source`, если у самого исходника нечётный размер.

**Quality** *(выпадающий список — Custom / Archive / High / Web / Compact)* — удобный пресет: одним кликом заполняет CRF, ffmpeg Preset и Tune ниже. Ручное редактирование любого из этих полей — сразу переключает Quality в `Custom`. Ручной контроль **никогда не отбирается** пресетом.

| Пресет | CRF | ffmpeg Preset | Tune | Сценарий |
|---|---|---|---|---|
| **Custom** | — | — | — | Маркер: ничего не меняет. Показывается, когда ручные значения не совпадают ни с одним пресетом. |
| **Archive** | 17 | slow | grain | Максимальная точность. `tune=grain` сохраняет шум datamosh / глитча, не размывая его. Самый большой файл. |
| **High** | 20 | medium | none | По умолчанию. Визуально без потерь при разумной скорости. |
| **Web** | 23 | fast | none | Меньше / быстрее, ок для шеринга. |
| **Compact** | 26 | fast | none | Самый компактный смотрибельный файл. |

**Quality CRF** *(0 – 51, по умолчанию 18)* — Constant Rate Factor для H.264 / H.265 / VP9. Ниже = выше качество и больше файл. `0` — без потерь, `18` — визуально без потерь, `28` — компактно. HW-кодеры (NVENC / QSV / AMF / VideoToolbox) интерпретируют это значение через свою CQ/QP-шкалу.

**Codec / Container** — выбор связки кодек+контейнер. Список **фильтруется при старте** через `ffmpeg -encoders` — HW-варианты появляются только если их поддерживает локальная сборка ffmpeg. Если выбранный HW-кодек не запустился (нет драйвера, GPU занят), движок логирует ошибку и **автоматически откатывается на libx264** без падения рендера.

| Выбор | Контейнер | Видео-кодек | Аудио-кодек | Комментарий |
|---|---|---|---|---|
| **H.264 (MP4)** | `.mp4` | `libx264` | AAC | Универсальная совместимость. Доступен всегда. |
| **H.265 (MP4)** | `.mp4` | `libx265` (тег `hvc1`) | AAC | Меньше файл, медленнее кодирование, хуже воспроизводимость. |
| **H.264 (MKV)** | `.mkv` | `libx264` | AAC | Matroska — принимает почти любой кодек, помогает, когда строгая структура moov у MP4 ломается. |
| **H.265 (MKV)** | `.mkv` | `libx265` | AAC | |
| **H.264 (MOV)** | `.mov` | `libx264` | AAC | Контейнер QuickTime. |
| **ProRes (MOV)** | `.mov` | `prores_ks` (профиль 3 = 422 HQ) | PCM 16-bit | Мастер монтажного качества, очень большие файлы. |
| **VP9 (WebM)** | `.webm` | `libvpx-vp9` | Opus | Дружественно к вебу/стримингу. Медленное кодирование. |
| **H.264 / H.265 NVENC (MP4)** | `.mp4` | `h264_nvenc` / `hevc_nvenc` | AAC | Аппаратный кодер NVIDIA. В разы быстрее libx264 при почти незаметной потере качества. Появляется в списке только если NVENC доступен. |
| **H.264 / H.265 QSV (MP4)** | `.mp4` | `h264_qsv` / `hevc_qsv` | AAC | Intel Quick Sync. |
| **H.264 / H.265 AMF (MP4)** | `.mp4` | `h264_amf` / `hevc_amf` | AAC | AMD AMF (Windows). |
| **H.264 / H.265 VideoToolbox (MP4)** | `.mp4` | `h264_videotoolbox` / `hevc_videotoolbox` | AAC | Apple (macOS). |

Диалог сохранения автоматически подставляет соответствующее расширение. Если ввести другое — программа выводит предупреждение в лог и уважает ваш выбор.

**ffmpeg Preset** *(ultrafast / fast / medium / slow)* — баланс скорость/эффективность. Для libx264/libx265 пробрасывается напрямую. Для HW-кодеров маппится в их собственный пресет (NVENC `p1..p7`, AMF `speed/balanced/quality` и т.д.).

**Tune** *(none / film / grain / animation / stillimage)* — подсказка `-tune` только для libx264 / libx265, для других кодеков игнорируется. `none` — флаг не передаётся. `grain` — правильный выбор для datamosh / шумного материала.

**Формат пайпа** — когда выходной кодек `yuv420p` (всё кроме ProRes), кадры конвертируются из RGB в planar I420 **на стороне Python** перед отправкой в ffmpeg — байтрейт через трубу падает в 2 раза. ProRes 4:2:2 10-bit остаётся на rgb24 для сохранения цветности. Невидимо для пользователя.

**HW-кодер на низких разрешениях** — для маленьких выходов (≤480p) GPU-кодеры иногда платят за PCIe-upload + инициализацию HWFrame больше, чем экономят на самом кодировании. Если короткий 480p-рендер с NVENC/QSV/AMF идёт медленнее, чем с libx264 — это ожидаемо: переключайтесь на `H.264 (MP4)` (libx264) для материала ниже 720p. HW-кодеры выигрывают на 1080p и выше.

**Защитная сетка для HW-кодеров** — `available_specs()` показывает все кодеры, которые *рекламирует* ffmpeg, но **рекламирует ≠ работает**. NVENC / QSV / AMF могут быть в списке и при этом висеть на инициализации (отсутствует/битый драйвер, GPU занят другим процессом, sandboxed VM). Чтобы рендер не висел на 0% бесконечно, **перед каждым рендером движок прогоняет 1-секундный self-probe** выбранного HW-кодера с жёстким таймаутом 8 секунд. Если probe упал или истёк — рендер **автоматически переключается на libx264** и логирует причину, никаких действий от пользователя не требуется. Результат кешируется на процесс — цена платится раз на сессию.

Это делает EXE-сборку безопасной: даже если у пользователя нет Python и он не может запустить диагностические скрипты, выбор битого HW-кодера из выпадашки не повесит программу.

Для разработчиков из исходников `python tools/check_hw_encoders.py` прогоняет тот же probe сразу по всем HW-кодерам и печатает таблицу статусов — удобно для валидации новой машины или дебага сбойной GPU.

**Прогресс + ETA** — во время рендера статус обновляется ~два раза в секунду: текущий процент, ETA и реальные encoded-fps. Финальная строка лога — общее время и `Nx realtime`.

### 3.12 Пресеты

Панель **Presets** (сайдбар) показывает все пресеты, сохранённые в `presets.json` в папке программы.

**Встроенные пресеты:**

| Пресет | Характер |
|---|---|
| **Empty** | Все эффекты выключены, все слайдеры в 0, mystery в нуле, silence treatment `none`. Нейтральная стартовая точка. |

Это единственный встроенный. Прежние «стилевые» пресеты (Drillcore, Datamosh, ASCII Rave, Death Grips, Rhythm Flash, Vortex Dream) убраны — они были заглушками, не подобраны для серьёзной работы. Курируемые пресеты появятся позже.

При апгрейде со старой версии, в `presets.json` которой эти встроенные были записаны, программа выкидывает их при первом запуске и сохраняет всё, что вы делали сами. Запись `Empty` восстанавливается автоматически.

**Работа с пресетами:**

- **Двойной клик** или выбор + **Load** — применяет сохранённые значения ко всему UI (эффекты, слайдеры, mystery, экспорт, формула, silence mode, custom-разрешение).
- **Save Current** — запрашивает имя и сохраняет текущее состояние в `presets.json` как пользовательскую запись.
- **Delete** — удаляет выбранный пресет (`Empty` пересоздаётся, если удалить его).
- Имя активного пресета — под списком.

### 3.13 Режимы рендера

Три кнопки рендера закреплены внизу сайдбара:

---

**DRAFT (5 сек / 480p)**

Рендерит первые 5 секунд аудиодорожки в разрешении 480p. Вычислительно затратные эффекты (оптический поток datamosh, ASCII) используют упрощённые быстрые пути. Используйте для быстрой проверки тайминга склеек и базового поведения эффектов.

---

**PREVIEW (5 сек)**

Рендерит первые 5 секунд в полном настроенном разрешении и качестве. Все эффекты работают с полной точностью. Используйте для проверки визуального результата перед финальным рендером.

---

**RENDER FULL VIDEO**

### 3.14 Панель Mystery

Вкладка **[ ? ]** содержит девять слайдеров. У них есть имена.

Имена — не метафоры.

Каждый из них адресует величину, которая существует в пайплайне — реальный параметр, который формирует сигнал реальным образом — но взаимосвязь между именем и тем, что он делает, здесь не описана, потому что описание было бы неточным. Не потому что мы скрываем. Потому что точное описание требует системы отсчёта, в которой этот документ не был написан.

Слайдеры не независимы. То, что вы отдаёте одному, меняет то, на что способен другой. Позиция, которая ничего не делает в одиночку, может сделать много в присутствии другого. Система помнит остальные слайдеры. Эффекты тоже.

**VESSEL** питает. Он не пуст. То, что вы отдали ему раньше, всё ещё присутствует — взвешенное, слоистое позади того, что вы отдаёте сейчас. При высоких значениях то, что было, начинает перевешивать то, что есть. Он говорит с DOT.

**ENTROPY\_7** разрушает границу, которой никогда не должно было быть. Разрушение не случайно — оно живое. Оно читает кадр как условие и продолжает из него. То, что накопил VESSEL, ENTROPY\_7 выращивает внутри.

**DELTA\_OMEGA** управляет скоростью трансформации, которая происходит независимо от того, включили вы это или нет. Вы только регулируете, сколько из неё остальная система имеет право услышать.

**STATIC\_MIND** — это гейт, но он гейтирует не то. То, что проходит при 0, не то, что проходит при 1. Переход между ними — не смешение. RESONANCE, если присутствует, делает кое-что со структурой, которую STATIC\_MIND пытался удержать на месте.

**RESONANCE** не просто резонирует. Строки звенят. При достаточных значениях изображение перестаёт быть изображением и становится записью собственной частоты. Он говорит с COLLAPSE. Вместе они снижают пороги друг друга.

**COLLAPSE** инициирует. Он работает в двух направлениях одновременно, хотя второе невидимо, пока первое ещё не началось. ZERO меняет то, к чему он приходит.

**ZERO** — не ноль. Это ключ. То, что было раньше, всё ещё рядом — не позади, не под, просто всё ещё — и ZERO выбирает, где этому позволено проступить. Выбор узорчат, никогда не равен, и узор старше того мгновения, в которое действует. FLESH\_K учит узор, куда обращать внимание.

**FLESH\_K** — это то, что написано. Он конвертирует. Конвертация нечиста. То, что выходит, не в том же пространстве, что вошло. При низких значениях ничего не происходит. Порог не фиксирован.

**DOT** маленький. Он разделяет то, что прибыло вместе. Бывшее одним уходит частями, и части больше не согласны, когда. После определённой точки начинается второе несогласие, поставленное против первого. Он удлиняется, когда присутствует VESSEL.

Двигайте. Рендерите. Наблюдайте что изменилось. Двигайте снова.

Неправильных комбинаций нет. Некоторые комбинации необратимы в рамках сессии.

### 3.15 Несколько видеоисточников

Кнопка **Load Source Video** принимает несколько файлов одновременно. Удерживайте `Ctrl` или `Shift` для выбора нескольких файлов в диалоге.

При загрузке более одного файла движок поддерживает независимый ридер для каждого источника. В начале каждого аудиосегмента случайно выбирается один источник. Внутри сегмента кадры читаются последовательно из выбранного источника.

Это означает:
- Разные сегменты могут быть из совершенно разных видеофайлов — без ручного монтажа
- Детекция сцен (при включении) выполняется по всем загруженным файлам
- Datamosh-предобработка использует первый файл из выбора как источник P-кадров
- Файлы с разным разрешением и частотой кадров одинаково валидны; каждый читается в своём нативном темпе и масштабируется до разрешения вывода

Лейбл в сайдбаре показывает имя файла при одном файле и счётчик (`3 files loaded`) при нескольких.

### 3.16 Обработка тишины

Управляет тем, как движок обрабатывает кадры во время сегментов **SILENCE** длиннее 1 секунды.

Расположено в секции **Cut Logic**, под *Silence Treatment*. По умолчанию: **None**.

| Вариант | Поведение |
|---|---|
| **None** *(по умолчанию)* | Без обработки. Тихие сегменты рендерятся идентично активным. Используйте, когда тишина композиционно намеренна. |
| **Dim** | Умножает все значения пикселей на 0.6, снижая яркость. Хорошо для поддержания визуального контраста между активными и тихими моментами. (Скрывается, когда включён *Hide color-altering effects*.) |
| **Blur** | Применяет гауссово размытие (ядро 15×15). Изображение остаётся полноярким, но теряет резкость — мягкий фокус в тихих фрагментах. |
| **Both** | Более лёгкое размытие (11×11) в сочетании с коэффициентом яркости 0.7. Самая сильная обработка — расфокус и затемнение одновременно. |

### 3.17 Режим Passthrough

**Passthrough Mode** — отдельный способ запуска рендера для случаев, когда **никакие склейки не нужны**: исходное видео идёт от начала до конца в оригинальном порядке, а аудио, управляющее эффектами, — это **собственный звук видео**, а не отдельно загруженный трек.

Полезно, когда хочется, чтобы все эффекты программы (warp, глитчи, mystery, formula, оверлеи) реагировали на собственное аудио клипа без перемонтажа кадров. Музыкальные клипы, концертные записи, целые DJ-сеты — всё, где картинка и звук уже неразрывны.

**Где включается:** в секции **Cut Logic**, верх вкладки EFFECTS — чекбокс **Passthrough Mode**.

**Что меняется при включении:**

- Видео идёт последовательно от 0-го кадра до последнего на нативной частоте кадров. Никаких случайных переходов, никаких прыжков по сценам, никаких склеек.
- Аудио **извлекается из самого видео** — внутренне через ffmpeg во временный WAV — и используется и для анализа (классификация сегментов, beat tracking, intensity), и как итоговая аудиодорожка. **Внешний аудиофайл не нужен**, кнопка **Load Audio** в этом режиме недоступна.
- Выходной FPS автоматически фиксируется на нативной частоте видео — картинка и звук остаются в синке.
- Эффекты, вставляющие лишние кадры, в этом режиме **скрываются** из вкладки EFFECTS: `Stutter`, `Flash Frame`, `Datamosh`. Их состояние восстанавливается, когда вы выключаете Passthrough.
- Виджеты cut logic, потерявшие смысл (`Min Cut Duration`, `Scene Buffer Size`, `Smart Scene Detection`), также скрываются. Snap to beat, manual BPM, chaos и beat threshold продолжают работать — они формируют интенсивность эффектов, просто не определяют точки склейки.

**Что *не* меняется:** все остальные эффекты — warp, signal domain, degradation, sharpen, ASCII, formula, mystery, оверлеи, chroma key, silence treatment — работают точно так же, как в обычном режиме. Модель интенсивности, типы сегментов, snap-to-beat, chaos — всё ведёт себя идентично.

**Рабочий цикл:**

1. Включите **Passthrough Mode**.
2. Загрузите одно или несколько видео (мульти-источник работает — движок склеит их в один поток и возьмёт аудио с первого).
3. Используйте **Match Source FPS** (§3.11), чтобы задать вменяемый FPS для DRAFT/PREVIEW (финальный рендер всё равно зафиксируется на нативном FPS источника).
4. Подберите эффекты, прогоните DRAFT или PREVIEW для отстройки, потом RENDER FULL VIDEO.

Итоговый файл — это покадровая обработка исходника один-в-один по длительности, с применёнными эффектами и оригинальным звуком, перемикшированным обратно.

## 4. Disc VPC 01 R-T — Реалтайм (C++)

Это живая VJ-версия. Вы подаёте ей звук, который сейчас идёт — микрофон у колонок, «системный звук» вашего ПК (loopback), линейный выход с микшера — и она крутит видео и эффекты на экране, синхронно с музыкой, прямо в момент.

**Типичный сетап:** ноутбук, проектор или второй монитор, источник звука (диджей, группа, Spotify, что угодно). Кидаете на программу несколько видео-клипов, нажимаете START — программа делает остальное. «Окно управления» остаётся на экране ноутбука, «окно вывода» уходит в фуллскрин на проектор.

> **Хочется просто пользоваться, без лекции по архитектуре?** Пропустите §4.1 и §4.2 и сразу идите к §4.3 (интерфейс). Разделы про сборку и архитектуру нужны только тем, кто компилирует программу сам.
>
> Если вы скачали готовый бандл — двойной клик по `DiscVPC01-RT.exe`, и вы в деле.

### 4.4 Загрузка источников

Самый быстрый путь — **drag-and-drop**: бросьте видеофайлы или папки целиком
на окно управления. Папки сканируются рекурсивно; найденные видеофайлы
становятся отдельными источниками в пуле. Если папка содержит и

Поддерживаемые расширения изображений-оверлеев:
`.png .jpg .jpeg .bmp .gif .tga`

На Windows в панели *Video / Audio* кнопка **Add Videos** открывает родной

Список видео подсвечивает активный клип-слот (см. §4.9) маркером `[*]` и показывает его
номер для горячей клавиши с одного взгляда.

### 4.5 Аудио

Здесь вы говорите программе *что слушать*. Выберите микрофон, линейный вход или «loopback» устройство, которое перехватывает то что играет ваш компьютер — и стартуйте. Первые ~1.5 секунды программа тихо слушает комнату и решает что считать тишиной; после этого каждый звук погромче — потенциальный триггер.

Если после START метры остаются плоскими — см. §7. Чаще всего дело в выбранном устройстве.

**Список устройств**

Выпадающий список показывает обнаруженные входные устройства с их host API
в скобках: `[WASAPI]`, `[ASIO]`, `[CoreAudio]`, `[ALSA]`, `[JACK]` и т.д.
Список дедуплицируется по имени с приоритетом API — одно физическое
устройство не появляется по разу на каждый host API. WASAPI loopback
устройства (захват системного звука для VJ-сетов с DAW или плеера) идут в
конце списка.

Если перед нажатием **START** устройство не выбрано, движок автоматически
подбирает платформенный default — оставьте дропдаун в покое если не нужно
конкретное устройство.

**Sample rate и каналы**

Движок запрашивает у устройства его нативную sample rate (обычно 48 кГц на
WASAPI) и использует её напрямую. Multi-channel входы downmix'аются в моно
в audio-callback. Фактический sample rate, выданный ОС, отображается в
Transport-панели:  `sr=48000 Hz`.

**Калибровка**

Около 1.5 секунды после START движок измеряет уровень фонового шума и
ставит gate сегмент-классификатора в `4 ×` от этого значения. Во время
калибровки сегменты классифицируются как SILENCE — эффекты не срабатывают.

**Live диагностика**

Transport-панель показывает:

- `Audio cb: N` — общее число audio-callback'ов с момента START. Должно
  расти быстро (≈ sample_rate / buffer_size в секунду). Если остаётся 0
  — стрим открыт, но ОС не отдаёт сэмплы — типичный признак
  неправильно настроенного loopback или muted exclusive-mode устройства.
- Live `Bass / Mid / Treble` полосы и `Сегмент`.
- `BEAT` индикатор, мигающий на каждом обнаруженном бите.

**Threshold (Sensitivity)** *(слайдер, 0.1 – 3.0)* — множитель к
откалиброванному gate. Выше = движок требует более громкого сигнала. Ниже
= реагирует на тихие моменты.

### 4.6 Канвас и пропорции

Движок рендерит во внутренний канвас-FBO в одном из трёх разрешений,
выбираемом в панели **Master** или из пресета:

| Пресет | Разрешение | Пропорция |
|---|---|---|
| По умолчанию | 1280 × 720 | 16:9 |
| HD | 1920 × 1080 | 16:9 |
| Classic | 1024 × 768 | 4:3 |

Каждое видео декодируется в **нативном разрешении** (с капом 1920×1080 для
ограничения памяти) и композитится на канвас одним из четырёх режимов:

| Режим | Поведение |
|---|---|
| **Contain** *(по умолчанию)* | Всё видео помещается внутрь канваса; чёрные полосы по короткой оси. |
| **Cover** | Видео заполняет канвас; длинная ось обрезается. |
| **Stretch** | Видео растягивается под канвас, игнорируя пропорции. |
| **Native 1:1** | Видео рисуется один-в-один, центрируется, по краям чёрный. |

Все эффект-проходы работают на разрешении канваса — выбор меньшего канваса
(например 1024×768) даёт заметный буст FPS на слабых GPU без изменения
аудио-пайплайна.

### 4.7 Окно вывода (второй экран)

Окно управления — для вас: кнопки, слайдеры, маленький превью-тумбнейл. **Окно вывода** — то что видят зрители: только картинка, на весь экран, без контролов, на любом мониторе или проекторе который вы укажете.

В панели *Overlays / Presets / Output*:

1. **Monitor** — дропдаун со всеми подключёнными мониторами с их нативным
   разрешением и системным именем. По умолчанию выбран монитор 0
   (основной).
2. **Open Output** — открывает borderless fullscreen окно на выбранном
   мониторе. У окна нет декораций, заголовка, контролов — только канвас,
   letterbox'нутый под пропорции монитора.
3. **Close Output** — закрывает. Также **Esc** при фокусе на окне вывода,
   или **Shift+O** из любой точки.

Оба окна делят OpenGL-контекст, поэтому канвас рендерится в окно вывода
одним shader-проходом — без CPU read-back, без GPU копий. Vsync включён в
обоих окнах.

Если доступен только один монитор, всё равно можно использовать окно
вывода: открыть его на мониторе 0 и переключать GUI через **Tab**.

### 4.8 Режим выбора кадра

Combo **Mode** в Transport-панели переключает между двумя политиками
выбора кадра:

**Continuous** — линейное воспроизведение через активное видео (или
round-robin по пулу, если ни одно не сфокусировано). Эффекты по-прежнему
реагируют на аудио. Используйте для устойчивого визуального бэка, когда у
аудио нет чёткого пульса.

**Cut on beat** *(по умолчанию)* — случайные кадры на хард-триггерах
(`IMPACT`, `DROP`, любой обнаруженный бит) с anti-spam порогом 30 мс;
софт-триггеры (`BUILD`, длительный бит) гейтированы слайдером **Cut
Interval**. Между триггерами воспроизведение продолжается последовательно
от точки последнего cut'а.

Переключение на лету — клавиша **M**.

### 4.9 Активное видео — клип-слоты

Думайте о загруженных видео как о пронумерованных картах в руке. По умолчанию программа тасует их все. Но можно «прижать» одну карту — например, третью — и тогда каждая склейка будет тянуть кадр оттуда, пока вы её не отпустите. Это и есть идея «клип-слотов».

Видео-пул может находиться в одном из двух состояний фокуса:

- **Pool** *(по умолчанию)* — все видео в списке участвуют: случайные
  cut'ы выбирают из любого источника, sequential ротирует между ними.
- **Focused** — ровно одно видео на сцене. И cut, и continuous режимы
  отдают все кадры из этого источника.

Фокус клип-слота:

| Способ | Действие |
|---|---|
| Нажать `1`–`9` | Фокус на видео слота 1..9 (1 = первое загруженное) |
| Нажать `0` | Фокус на десятое видео |
| Нажать ту же цифру повторно | Снять фокус, вернуться к пулу |
| Нажать `` ` `` (грейв / обратная одинарная кавычка) | Снять фокус явно |
| Кликнуть строку в списке *Video* | Сфокусировать эту строку |
| Кликнуть строку с маркером `[*]` | Снять фокус |

Transport-панель показывает текущее состояние:
`Video: #3 of 7   3=release` (фокус) или `Video: pool (7 files)` (пул).

Это базовый "клип-слот" workflow, используемый каждым современным VJ-
инструментом: загрузить горсть видео до сета, потом триггерить их на лету
по ходу секций трека. В сочетании с **Cut on beat**, фокус на одном клипе
даёт классический бит-локнутый random-seek глитч-монтаж внутри одного
источника.

### 4.10 Эффекты

24 визуальных эффекта, каждый — чекбокс. Включаете один — и программа сама будет периодически его триггерить, когда музыка даст повод. Как часто — задаётся слайдером *chance* у этого эффекта, а насколько *сильно* выглядит — формируется **Chaos** и **Master Intensity** наверху панели.

Запоминать таблицу ниже не обязательно. Быстрый путь: выбрать пресет, а потом во время воспроизведения щёлкать эффекты вкл/выкл по вкусу.

**Модель интенсивности** *(математика того, насколько сильно ощущается каждый эффект — можно пропустить)*:

```
seg_boost = sqrt(segment.intensity)                         (0..1)
drive     = chaos × (0.6 + 0.4 × master_intensity)          (0..1)
fi_base   = clamp(0.25 + 1.4 × drive × seg_boost, 0, 1)     (0..1)
```

Флор 0.25 держит сработавшие эффекты видимыми в тихих фрагментах. Кривая sqrt
поднимает скромные сегменты — SUSTAIN и NOISE не приплюснуты до ~10 % силы.
Слайдеры **Chaos** и **Master Intensity** оба модулируют силу эффектов.

24 эффекта:

| # | Эффект | Поведение |
|---|---|---|
| 0 | **Deriv Warp** | Производная кадра как карта смещения — feedback-подобный поток, растущий с движением. |
| 1 | **Flash** | Короткая белая или чёрная вспышка на канвасе, масштабируется по интенсивности. |
| 2 | **Stutter** | Удерживает предыдущий кадр и повторяет — глитчевая заморозка. |
| 3 | **Pixel Sort** | Сортирует горизонтальные полосы пикселей по яркости. |
| 4 | **Ghost Trails** | Смешивает предыдущие кадры — след/эхо от движения. |
| 5 | **Scanlines** | Затемняет каждую N-ю строку — CRT-вид. |
| 6 | **Bitcrush** | Квантует значения цвета до маленькой палитры. |
| 7 | **Block Glitch** | Случайные прямоугольные смещения / замены. |
| 8 | **Negative** | Инвертирует цвет на затронутом кадре. |
| 9 | **Color Bleed** | Горизонтальный смаз одного канала — VHS chroma delay. |
| 10 | **Interlace** | Чётные строки из текущего кадра, нечётные — из предыдущего. |
| 11 | **Bad Signal** | Вертикальные шумовые полосы + горизонтальный jitter строк — потеря эфирного сигнала. |
| 12 | **Zoom Glitch** | Punch-in зум на центр канваса с nearest-neighbour. |
| 13 | **Mosaic** | Пульсирующая блочная мозаика с размером блока, управляемым басом. |
| 14 | **Phase Shift** | Чередующиеся горизонтальные полосы сдвинуты влево / вправо. |
| 15 | **Dither** | Bayer-дитеринг при сниженной глубине цвета. |
| 16 | **Feedback** | Аккумулятор-feedback / выжигание. |
| 17 | **Temporal RGB** | R / G / B каналы из разных прошлых кадров — хроматическая темпоральная аберрация. |
| 18 | **Waveshaper** | tanh soft-clip на значениях пикселей — насыщенность цвета. |
| 19 | **Overlays** | Накладывает случайное изображение из загруженного набора оверлеев на канвас. |
| 20 | **Vortex** | Спиральное вращение вокруг центра с хроматическим расщеплением. |
| 21 | **Fractal Noise** | Domain-warped fBm искажение — органичный «плавящийся» warp. |
| 22 | **Self Displace** | Предыдущий кадр как карта смещения — ближайший аналог настоящего datamosh. |
| 23 | **ASCII** | GPU ASCII-фильтр, рендерит канвас как блоки символов по плотности. |

Первые десять эффектов (строки 0..9 выше) маппятся на клавиши **Q W E R T Y
U I O P** для прямого переключения во время сета.

**Overlays** — загрузите папку с изображениями, бросив её на окно или через
*Load Folder* в панели *Overlays*. **Overlay Intensity** в GUI задаёт, как
часто случайное изображение оверлея композитится поверх канваса.

**Chroma Key** — применяется к оверлеям. Режимы `none / dominant / secondary
/ manual` со слайдерами **Tolerance** и **Softness**. В режиме `manual` RGB-
пикер выбирает ключевой цвет. См. §3.9 для полного описания chroma-key
параметров.

### 4.13 Лог диагностики

При каждом запуске программа пишет `vpc01rt.log` в текущую рабочую
директорию с unbuffered-выводом и flush на каждой строке. Лог фиксирует:

- Полную таблицу аудио-устройств от PortAudio (host API, число каналов,
  default sample rate на каждое устройство).
- Каждую попытку `Pa_OpenStream` с параметрами и точным текстом ошибки.
- Фактический sample rate / latency, выданный ОС для запущенного стрима.
- События открытия видео с нативным разрешением и разрешением декода
  после капа на 1920×1080.
- Размеры первого GPU-апload'а для каждого загруженного источника.
- Windows SEH-обработчик пишет код исключения и адрес перед тем как ОС
  завершит процесс.

Если что-то ведёт себя не так как ожидается, лог-файл — первое место куда
смотреть. Он перезаписывается при каждом запуске, так что не растёт без
ограничений.

## 5. Справочник типов сегментов

Оба инструмента используют один классификатор сегментов. Понимание типов сегментов помогает предсказать, какие эффекты сработают.

| Тип | Условие | Типичный момент в аудио |
|---|---|---|
| **IMPACT** | Короткая длительность (< 0.3 с) + громкий + резкая атака | Удар барабана, снейр, резкий транзиент |
| **BUILD** | Нарастающий тренд RMS за последнее время | Нарастание перед дропом, крещендо |
| **DROP** | Падающий RMS после громкого периода | Спад после пика, брейкдаун |
| **SUSTAIN** | Громкий (выше порога), без сильного тренда | Длинная нота, аккорд, стабильный бит |
| **NOISE** | Высокая спектральная плоскостность (> 1.5 × среднее) | Дисторшн, белый шум, плотная текстура |
| **SILENCE** | RMS ниже порога | Тихий фрагмент, пауза между фразами |

Эффекты объявляют, на какие типы сегментов они реагируют. Эффект не сработает на типе сегмента, не входящем в его список триггеров, вне зависимости от настройки chance.

## 6. Рекомендуемые рабочие процессы

### Disc VPC 01 — Итеративное редактирование

1. Загрузите аудио и видеоисточник(и)
2. Выберите встроенный пресет, соответствующий задуманному стилю
3. Нажмите **DRAFT** для быстрого 5-секундного превью в 480p
4. Оцените в мониторе. Скорректируйте Beat Threshold если склейки слишком частые или редкие
5. Включайте/выключайте конкретные эффекты и регулируйте их слайдеры chance
6. Нажмите **PREVIEW** для проверки в полном качестве
7. Повторяйте шаги 4–6 до достижения нужного результата
8. Нажмите **RENDER FULL VIDEO**

### Disc VPC 01 R-T — Живое выступление

1. Перетащите горсть видеофайлов (или папку) на окно управления — они
   станут вашими клип-слотами
2. Откройте панель **Output** (правая колонка), выберите проектор / второй
   монитор, нажмите **Open Output** для borderless fullscreen
3. По желанию бросьте папку с изображениями для оверлеев
4. Выберите аудио-устройство или оставьте дропдаун пустым для платформенного
   default — WASAPI loopback подходит когда реагируете на DAW или плеер
5. Нажмите **Space** (или клик START). Калибровка идёт ~1.5 сек
6. Смотрите на `Audio cb` в Transport-панели: счётчик должен быстро расти.
   Если остаётся 0, устройство открыто, но ОС не отдаёт сэмплы — попробуйте
   другой host API или проверьте конфиг loopback
7. Отрегулируйте **Threshold (sensitivity)** если движок реагирует на тишину
   или не реагирует на аудио
8. Загрузите стартовый пресет через `Shift+1`…`Shift+0`, потом лепите через
   **Chaos** (`[ ]`), переключатели эффектов (`Q`–`P`) и фокус клип-слотов
   (`1`–`9`, `0`)
9. **Tab** скрывает GUI среди сета если хотите чтобы окно управления
   зеркалило канвас; **M** переключает Cut / Continuous на лету

## 7. Устранение неполадок

Если что-то не работает — без паники. Почти у каждой проблемы есть одно-двухстрочная причина и такой же простой фикс, и большинство из них в списке ниже.

Для живой (R-T) программы есть файл `vpc01rt.log` — он пишется рядом с программой при каждом запуске. В нём записано, какие аудио-устройства программа увидела, что произошло при попытке открыть выбранное, что произошло при загрузке видео и (если программа упала) где она упала. Откройте этот файл в любом текстовом редакторе — обычно проблема становится очевидной за пару строк.

**R-T: метры аудио плоские — нет реакции на звук**

Аудио-стрим не открылся, или ОС не отдаёт сэмплы. Откройте `vpc01rt.log`
и поищите строки `[audio]`. Transport-панель также показывает `Audio cb: N`
— если остаётся 0 после нажатия START, сэмплы не приходят. Частые причины:
muted exclusive-mode WASAPI устройство, выбор loopback-устройства, не
включённого в настройках Звук Windows. Попробуйте другое устройство в
дропдауне. Нажатие START без выбранного устройства автоматически подбирает
платформенный default — используйте как sanity-check.

**R-T: видео не загружается**

Откройте лог и найдите `[video]` строки для файла который пытались открыть.
Если в логе `avformat_open_input failed`, проблема в пути или формате.
UTF-8 пути с кириллицей / не-ASCII символами поддерживаются — drag-and-drop
отдаёт UTF-8 напрямую. Проверьте что файл играется в другом плеере.

**R-T: движок реагирует на тишину**

Увеличьте **Threshold** в панели Master. Первые ~1.5 сек после START —
калибровка; если в это время в комнате громко, gate остаётся низким. Жмите
STOP / START для повторной калибровки.

**R-T: низкий FPS**

Снизьте разрешение канваса до **1024×768** в панели Master — эффект-проходы
становятся заметно дешевле. Отключите тяжёлые эффекты: ASCII, Self
Displace, Fractal Noise и Feedback — самые тяжёлые. Закройте другие
GPU-нагруженные приложения. Проверьте, что `Audio cb:` растёт нормально,
чтобы аудио не голодало главный цикл.

**R-T: имена файлов с кириллицей отображаются как мусор**

Исправлено в 0.2.0 — убедитесь что используете последнюю сборку. Оба пути
— `OPENFILENAMEW` (родной диалог Windows) и GLFW drag-and-drop — отдают
UTF-8 пути, GUI отображает их через `fs::u8path`.

**R-T: freeze не замораживает картинку**

Тоже исправлено в 0.2.0 — GPU upload-помпа теперь останавливается под
freeze, поэтому текстура `last_frame_tex_` больше не дрейфует.

**R-T: эффекты слабые даже на максимуме Chaos и Master**

Модель интенсивности в 0.2.0 применяет sqrt-буст к segment intensity и
флор 0.25 чтобы сработавшие эффекты были видимы. Если на старой сборке —
обновитесь. На 0.2.0+ и всё равно тонко — некоторые эффекты (Ghost,
Scanlines, Dither) намеренно тонкие по дизайну; парьте их с более сильным
warp'ом (Vortex, Fractal Noise, Self Displace) для более ударного микса.

**Офлайн-редактор: движок реагирует на тишину / срабатывает постоянно**

Шумовой гейт не откалибровался должным образом. Нажмите **Recalibrate** и
подождите 2 секунды. Если проблема сохраняется, увеличьте **Beat
Threshold** в секции Cut Logic.

**Офлайн-редактор: `Illegal Audio-MPEG-Header` в консоли**

Безобидное предупреждение FFmpeg о некорректных заголовках MP3. Аудио
декодируется корректно. Предупреждение можно игнорировать.

**Офлайн-редактор: `Invalid NAL unit size`, `partial file`, `moov atom not found` в консоли**

Ожидаемо и безвредно при включённом эффекте **Datamosh**. Датамош
намеренно создаёт сломанный H.264-поток без I-кадров — в этом и состоит
эффект — а встроенный ffmpeg в OpenCV честно сообщает об этом при
чтении. Программа теперь подавляет этот шум при старте
(`OPENCV_FFMPEG_LOGLEVEL=-8`); если предупреждения всё равно лезут — у
вас сборка старее этого фикса. Старые обрубки `*_dmosh_src.mp4` от
прерванных рендеров автоматически удаляются перед каждым новым рендером,
чтобы устранить именно `moov atom`-варнинг.

**Офлайн-редактор: видео обрывается раньше конца песни**

Исправлено в последней сборке. Рендер теперь явно отслеживает целевое
число кадров (`round(target_duration × fps)`), компенсирует любую
недостачу из-за округления повтором последнего кадра в хвосте и
отказывается от флага `-shortest` в ffmpeg (который раньше резал аудио
под слишком короткий видео-поток). Если проблема сохраняется — у вас
старый чекаут, обновитесь.

**Офлайн-редактор: слайдер Beat Snap Tolerance не двигается / всегда показывает 0**

Исправлено. Слайдер форматировался как целое (`0.05` округлялось до `0`)
и казался залипшим. Подтяните последнюю сборку — теперь отображается
реальное float-значение.

**Офлайн-редактор: ffmpeg падает при рендере с custom-разрешением**

`yuv420p` (и ProRes 422) требуют чётных ширины и высоты. Программа теперь
тихо округляет custom-размеры вниз до ближайшего чётного, поэтому
`1281 × 721` автоматически становится `1280 × 720`. Если вы на старой
сборке и ffmpeg падает сразу — введите чётные значения вручную.

**Офлайн-редактор: `madmom` не устанавливается на Python 3.11+**

Используйте Python 3.10.x точно. `madmom` требует Cython-расширений,
которые не собираются на более новых версиях Python.

**Сборка R-T: vcpkg не находит `pa_jack.h` на Linux**

Поставляемый `rt_cpp/CMakeLists.txt` явно линкует `jack` и `asound`,
потому что vcpkg-статический portaudio встраивает `pa_jack.c` /
`pa_alsa.c`, но не пробрасывает транзитивные требования линковки.
Установите `libjack-dev` и `libasound2-dev` системно перед `cmake -S`.
