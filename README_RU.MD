## 1. Обзор

**Disc VPC 01** — офлайн-инструмент для алгоритмического видеомонтажа. Анализирует аудиофайл, делит его на типизированные сегменты и собирает новое видео из исходного материала, применяя аудиореактивные склейки и визуальные эффекты. Результат экспортируется как видеофайл.

**Disc VPC 01 R-T** — реалтайм-версия. Слушает живой аудиовход (микрофон, линейный вход или системный loopback) и управляет воспроизведением видео и эффектами в режиме реального времени, кадр за кадром. Предназначен для живых выступлений и VJ-сетов.

Оба инструмента используют одно ядро аудиоанализа и одну библиотеку эффектов.


## 3. Disc VPC 01 — Офлайн-редактор

### 3.1 Интерфейс

Окно разделено на три вертикальные панели:

| Панель | Содержимое |
|---|---|
| **Левая (сайдбар)** | Загрузка файлов, пресеты, кнопки рендера |
| **Центральная** | Вкладки эффектов: EFFECTS / EXPORT / MYSTERY |
| **Правая** | Предпросмотр, прогресс-бар, лог |

### 3.2 Загрузка файлов

Перед рендером необходимо загрузить два файла:

**Load Audio (WAV / MP3)** — аудиодорожка, которая управляет всем анализом. Движок читает файл целиком до начала рендера. Поддерживаемые форматы: `.wav`, `.mp3` и любые форматы, читаемые `librosa`.

**Load Source Video** — видеоклип, из которого будут нарезаться и обрабатываться кадры. Принимается любое разрешение; разрешение выходного файла задаётся в настройках экспорта.

Зелёная точка (●) рядом с каждой кнопкой подтверждает успешную загрузку файла.

Дополнительно можно загрузить **папку с оверлеями** через вкладку Overlays (см. §3.9).

### 3.3 Аудиоанализ

При запуске рендера движок выполняет полный анализ аудиодорожки с помощью `librosa`:

1. **Детекция атак (Onset detection)** — выявляет транзиентные удары в сигнале (удары барабанов, атаки нот и т.д.)
2. **Отслеживание бита (Beat tracking)** — извлекает сетку битов для привязки к ритму
3. **Вычисление RMS** — рассчитывает среднеквадратичную энергию для каждого сегмента
4. **Спектральная плоскостность** — измеряет, насколько спектр «похож на шум» (белый шум = 1.0, чистый тон ≈ 0)
5. **Классификация сегментов** — присваивает каждому сегменту один из шести типов (см. §5)

Анализ выполняется один раз за сессию для каждого аудиофайла. Результаты кешируются в памяти для повторных рендеров.

### 3.4 Логика монтажа

Секция **Cut Logic** (первая аккордеонная группа во вкладке EFFECTS) управляет тем, как движок выбирает и выстраивает видеоклипы.

---

**Smart Scene Detection** *(вкл/выкл)*

При включении движок использует `PySceneDetect` для поиска границ сцен в исходном видео до начала рендера. Клипы выбираются с точек перехода, а не случайных кадров — это даёт более визуально связные переходы.

Рекомендуется для материала с отчётливыми сценами (кино, съёмочный материал). Отключите для абстрактного или повторяющегося видео.

---

**Global Chaos Level** *(0.0 – 1.0)*

Управляет общей случайностью выбора клипов. При `0.0` клипы выбираются предсказуемо на основе детекции сцен. При `1.0` каждая склейка выбирает случайный кадр независимо от сцен. Также масштабирует вероятность срабатывания большинства эффектов.

---

**Beat Threshold** *(0.5 – 2.0)*

Множитель, применяемый к среднему RMS для определения того, является ли сегмент «громким». Сегмент считается громким, если:

```
rms_сегмента > rms_среднее × threshold
```

Меньшие значения → больше сегментов считаются громкими → больше склеек и эффектов. Большие значения → только самые пиковые моменты вызывают реакцию.

**Рекомендуемые начальные значения:**
- Тихий или динамичный материал: `0.9 – 1.1`
- Плотный/компрессированный материал: `1.3 – 1.6`

---

**Transient Sensitivity** *(0.1 – 1.5)*

Порог для детекции резкой атаки (быстрый рост RMS). Используется для классификации сегментов IMPACT. Меньшие значения детектируют больше транзиентов; большие реагируют только на очень резкие удары.

---

**Min Cut Duration (sec)** *(0.0 – 0.3)*

Сегменты короче этого значения объединяются со следующим и не порождают склейку. Предотвращает микросклейки и мерцание на плотном аудио. Типичный рабочий диапазон: `0.04 – 0.1`.

---

**Scene Buffer Size** *(2 – 30)*

Количество уникальных временных меток сцен, хранящихся в памяти для случайного выбора в хаотичные моменты. Большой буфер даёт больше разнообразия; маленький заставляет монтаж циклически использовать меньше сцен.

---

**Snap Cuts to Beat Grid** *(вкл/выкл)*

При включении каждый обнаруженный атак привязывается к ближайшей позиции на сетке битов, если попадает в окно **Beat Snap Tolerance**. Это делает склейки более точными по ритму.

**Beat Snap Tolerance** *(0.01 – 0.15 сек)* — максимальное расстояние, на которое атак может быть сдвинут к ближайшему биту. Если атак дальше — остаётся на месте.

---

### 3.5 Эффекты — Основные

Основные эффекты — первичные визуальные трансформации, применяемые к каждому кадру. У каждого есть переключатель вкл/выкл и, где применимо, слайдер **chance** — вероятность срабатывания на каждом подходящем кадре.

---

**Stutter / Drill**

*Триггер: только сегменты IMPACT*

Повторяет первый микросегмент удара 2, 4 или 8 раз подряд, создавая эффект дрилла или заикания. Слайдера chance нет — эффект срабатывает на каждом IMPACT при включении.

---

**Flash Frame**

*Триггер: DROP, IMPACT*

Вставляет короткий белый или чёрный кадр (или смешение к белому/чёрному) в моменты резкого изменения энергии. Интенсивность вспышки масштабируется с intensity-значением сегмента.

**Chance** *(0.0 – 1.0)* — вероятность срабатывания на каждом подходящем сегменте.

---

**Ghost Trails**

*Триггер: SUSTAIN, BUILD*

Смешивает текущий кадр с предыдущим, создавая шлейф движения или эффект призрака. Подходит для медленных нарастаний и длинных сустейнов.

**Opacity** *(0.0 – 1.0)* — насколько сильно предыдущий кадр просвечивает. При `0.0` эффект невидим; при `1.0` изображение практически заморожено.

---

**Pixel Sort**

*Триггер: NOISE, IMPACT*

Сортирует столбцы пикселей в горизонтальных полосах по яркости, тону или насыщенности. Даёт характерную глитч-эстетику, напоминающую сбой кодека.

**Chance** *(0.0 – 1.0)* — вероятность на каждый подходящий сегмент.

**Pixel Sort Intensity** *(0.0 – 1.0)* — высота сортируемых полос. При низких значениях затрагиваются только тонкие полосы; при высоких — почти весь кадр.

**Sort Axis** — свойство пикселей, используемое как ключ сортировки:
- `luminance` — сортировка по воспринимаемой яркости (по умолчанию, наиболее читаемо)
- `hue` — сортировка по углу цвета в пространстве HSV (более красочный результат)
- `saturation` — сортировка по чистоте цвета

---

**Datamosh**

*Триггер: NOISE*

В режимах **Draft** и **Preview**: симулирует эффект растекания P-кадров с помощью оптического потока — векторы движения предыдущего кадра применяются для деформации изображения.

В режиме **Final render**: применяется реальное удаление I-кадров на уровне кодека, производя подлинный datamosh-эффект.

**Chance** *(0.0 – 1.0)* — вероятность на каждый подходящий сегмент.

---

**ASCII Filter**

*Триггер: SUSTAIN*

Конвертирует кадр в ASCII-арт, отрисованный обратно как изображение. Каждый символ представляет блок пикселей, плотность символа соответствует яркости блока.

**Char Size (px)** *(4 – 40)* — размер блока символа в пикселях. Меньшие значения дают более детальный результат; большие — более блочный, абстрактный вид.

**Blend (0 – 1)** — накладывает ASCII на исходный кадр. При `0.0` — чистый ASCII. При `1.0` — исходный кадр без изменений. Промежуточные значения смешивают оба.

**ASCII Color Mode:**
- `fixed` — все символы рисуются в заданном цвете переднего плана на заданном фоне
- `original` — каждый символ берёт средний цвет исходных пикселей, которые он представляет
- `inverted` — то же самое, но с инвертированными цветами

**ASCII FG/BG Color** — RGB-слайдеры (0–255) для переднего плана и фона в режиме `fixed`.

### 3.6 Эффекты — Глитч

Эти эффекты симулируют деградацию сигнала, артефакты кодека и ошибки передачи.

| Эффект | Триггер | Описание |
|---|---|---|
| **RGB Shift** | IMPACT, BUILD | Смещает красный канал вправо, синий влево на смещение, пропорциональное интенсивности. Хроматическая аберрация. |
| **Block Glitch** | IMPACT, DROP | Заменяет случайные прямоугольные блоки мусорными пикселями или копирует их с случайной позиции. |
| **Pixel Drift** | NOISE, IMPACT | Смещает целые строки пикселей вбок на случайные расстояния, имитируя потерю горизонтальной синхронизации. |
| **Color Bleed / VHS Smear** | NOISE | Размазывает цветовые каналы горизонтально, имитируя аналоговую задержку хромы. |
| **Freeze + Corrupt** | DROP | Замораживает кадр и применяет случайную порчу пикселей, как будто видеопоток завис. |
| **Negative** | IMPACT | Инвертирует все значения цветов (`255 - значение`) на один кадр. |

У каждого эффекта есть слайдер **Chance** — вероятность срабатывания на подходящем сегменте.

---

### 3.7 Эффекты — Сложные

Эти эффекты используют состояние нескольких кадров или более вычислительно затратные трансформации.

---

**Feedback Loop**

*Триггер: SUSTAIN, BUILD — срабатывает на каждом подходящем кадре при включении, без chance*

Смешивает текущий кадр с затухающей копией, накопленной из предыдущих кадров — эффект видеофидбэка. Накопление усиливает сильные сигналы и рассеивает тихие.

---

**Phase Shift** *(Chance)*

Делит кадр на горизонтальные полосы и смещает чередующиеся полосы влево и вправо на нарастающие смещения.

---

**Mosaic Pulse** *(Chance)*

Пикселизирует кадр путём уменьшения и увеличения масштаба. Размер блока управляется значением RMS баса — пикселизация пульсирует с низкочастотной энергией.

---

**Echo Compound** *(Chance)*

Накладывает несколько копий кадра на себя со смещёнными цветовыми оттенками и лёгким пространственным смещением — многослойное хроматическое эхо.

---

**Kali Mirror** *(Chance)*

Применяет калейдоскопическое зеркальное преобразование, отражая и поворачивая квадранты кадра для создания симметричных геометрических узоров.

---

**Glitch Cascade** *(Chance)*

Объединяет несколько глитч-эффектов в цепочку на одном кадре, суммируя их результаты. Конкретная цепочка рандомизируется для каждого кадра.

---

### 3.8 Эффекты — Деградация

Эффекты деградации симулируют изношенные или низкокачественные носители записи/воспроизведения.

| Эффект | Триггер | Описание |
|---|---|---|
| **Scan Lines** | SUSTAIN, NOISE | Рисует тёмные горизонтальные линии через строку, имитируя CRT-монитор. |
| **Bitcrush / Posterize** | Любой | Снижает глубину цвета, квантуя значения пикселей до меньшего числа уровней. |
| **JPEG Crush** | IMPACT, NOISE | Перекодирует кадр с очень низким качеством JPEG и декодирует обратно — блочные артефакты компрессии. |
| **Fisheye / Barrel** | BUILD | Применяет бочкообразное искажение линзы. |
| **VHS Tracking** | NOISE, DROP | Симулирует ошибку трекинга VHS-кассеты: горизонтальное смещение верху и внизу + шум яркости. |
| **Interlace** | SUSTAIN | Разделяет чётные и нечётные строки развёртки, смещая их на несколько пикселей. |
| **Bad Signal** | DROP, NOISE | Накладывает артефакты цифрового распада сигнала — горизонтальные полосы, цветовые сдвиги, шумовые полосы. |
| **Dithering** | SILENCE, SUSTAIN | Применяет упорядоченный паттерн дизеринга, сокращая эффективную цветовую палитру. |
| **Zoom Glitch** | IMPACT | Применяет резкий случайный зум для одного кадра — внезапный импульс прыжка внутри кадра. |

У каждого эффекта есть слайдер **Chance**.

---

### 3.9 Эффекты — Оверлеи и Chroma Key

Оверлеи накладывают внешние файлы изображений на видеокадры. Папку с оверлеями загружают через кнопку в сайдбаре или через вкладку Overlays.

**Enable Overlays** *(вкл/выкл + Chance)*

Движок выбирает случайное изображение из папки на каждом подходящем кадре и накладывает его на видео.

**Opacity** *(0.0 – 1.0)* — прозрачность изображения оверлея.

**Scale Max** *(0.05 – 1.0)* — максимальный размер оверлея как доля от размеров кадра.

**Scale Min** *(0.05 – 1.0)* — минимальный размер. Движок выбирает случайный масштаб между Min и Max для каждого появления.

**Blend Mode:**
- `screen` — осветляет изображение (хорошо для бликов, засветок, текстур)
- `normal` — стандартный альфа-композитинг поверх кадра
- `multiply` — затемняет изображение (хорошо для гранжа, зерна плёнки)

**Position:**
- `random` — оверлей размещается в случайной позиции в пределах кадра
- `center` — оверлей центрируется по кадру
- `corners` — оверлей циклически обходит угловые позиции

Поддерживаемые форматы: `.png`, `.jpg`, `.gif` (только первый кадр), `.bmp`, `.mp4`, `.avi`, `.mov`, `.mkv`, `.flv`, `.wmv`, `.mpg`, `.mpeg`

Видеофайлы загружаются покадрово; каждый кадр становится отдельным изображением оверлея в пуле.

---

### Chroma Key (Greenscreen / Хромакей)

Удаляет цвет фона из каждого изображения оверлея перед наложением, делая этот цвет прозрачным.

**Режим (Mode):**
- `none` — хромакей отключён (по умолчанию)
- `dominant` — автоматически определяет и удаляет наиболее часто встречающийся цвет в оверлее (например, зелёный экран)
- `secondary` — удаляет второй по частоте цвет
- `manual` — удаляет цвет, заданный RGB-слайдерами ниже

**Tolerance** *(5 – 60)* — ширина диапазона оттенков вокруг ключевого цвета, который вырезается. Увеличьте, если края зелёного экрана удаляются не полностью; уменьшите, чтобы не захватывать лишнее.

**Edge Softness** *(1 – 21)* — применяет гауссово размытие к краю маски ключа, сглаживая переход между вырезанными и невырезанными областями.

**Manual Key Color (R / G / B)** *(0 – 255 каждый)* — целевой цвет для режима `manual`. По умолчанию — чистый зелёный (0, 255, 0).

---

### 3.10 Настройки экспорта

Расположены во вкладке **EXPORT**.

**FPS** *(1 – 60, по умолчанию 24)* — частота кадров выходного файла. Не влияет на тайминг исходного аудио; контролирует только количество генерируемых кадров в секунду.

**CRF** *(1 – 51, по умолчанию 18)* — Constant Rate Factor для кодирования H.264/H.265. Меньшие значения = более высокое качество и больший размер файла. Значение `18` визуально почти без потерь. `23` — значение H.264 по умолчанию. Значения выше `30` дают заметные артефакты компрессии.

---

### 3.11 Пресеты

Панель **Presets** (сайдбар) предоставляет именованные наборы параметров для распространённых стилей.

**Встроенные пресеты:**

| Пресет | Характер |
|---|---|
| **Drillcore (Fast Cut/Stutter)** | Высокий хаос, быстрые склейки, тяжёлый stutter и block glitch, RGB shift |
| **Datamosh (P-frame Bleed)** | Исходник с детекцией сцен, P-frame datamosh, pixel sort, ghost trails, color bleed |
| **ASCII Rave** | Доминирующий ASCII-фильтр, flash, scanlines, stutter |
| **Death Grips Mode** | Максимальный хаос, минимальная длительность склейки, JPEG crush, bad signal, VHS, negative |
| **Rhythm Flash (Scene Mix)** | Детекция сцен, flash на каждом бите, zoom glitch |

**Работа с пресетами:**

- **Двойной клик** или выбор + **Load** — применяет все сохранённые значения параметров
- **Save Current** — запрашивает имя и сохраняет все текущие значения слайдеров/переключателей в `presets.json`
- **Delete** — удаляет выбранный пресет из `presets.json`
- Имя активного пресета отображается под списком

Пользовательские пресеты сохраняются между сессиями в `presets.json` в папке программы.

---

### 3.12 Режимы рендера

Три кнопки рендера закреплены внизу сайдбара:

---

**DRAFT (5 сек / 480p)**

Рендерит первые 5 секунд аудиодорожки в разрешении 480p. Вычислительно затратные эффекты (оптический поток datamosh, ASCII) используют упрощённые быстрые пути. Используйте для быстрой проверки тайминга склеек и базового поведения эффектов.

---

**PREVIEW (5 сек)**

Рендерит первые 5 секунд в полном настроенном разрешении и качестве. Все эффекты работают с полной точностью. Используйте для проверки визуального результата перед финальным рендером.

---

**RENDER FULL VIDEO**

Рендерит полный выходной ролик на всю длительность аудиодорожки. Прогресс-бар и лог статуса показывают прогресс в реальном времени. Путь к выходному файлу отображается в консоли по завершении. Диалог сохранения предлагает имя вида `disc_XXXX.mp4` (случайный четырёхзначный номер).

Кнопка **STOP** (правая панель) прерывает активный рендер или воспроизведение предпросмотра.

---

### 3.13 Панель Mystery

Вкладка **[ ? ]** содержит девять слайдеров. У них есть имена.

Имена — не метафоры.

Каждый из них адресует величину, которая существует в пайплайне — реальный параметр, реально влияющий на сигнал — но связь между именем и тем, что он делает, здесь не описана, потому что описание было бы неточным. Не потому что мы это скрываем. Потому что точное описание требует системы отсчёта, в которой этот документ не написан.

VESSEL питает. ENTROPY\_7 разрушает границу, которой никогда не должно было быть. DELTA\_OMEGA управляет скоростью трансформации, которая происходит независимо от того, включили вы это или нет — вы просто регулируете, сколько её видите. STATIC\_MIND — это гейт, но он гейтирует не то. RESONANCE не резонирует. COLLAPSE инициирует. ZERO — не ноль. FLESH\_K — это то, что написано. DOT маленький.

Двигайте. Рендерите. Наблюдайте что изменилось. Двигайте снова.

Неправильных комбинаций нет. Некоторые комбинации необратимы в рамках сессии.

---

### 3.14 Несколько видеоисточников

Кнопка **Load Source Video** принимает несколько файлов одновременно. Удерживайте `Ctrl` или `Shift` для выбора нескольких файлов в диалоге.

При загрузке более одного файла движок поддерживает независимый ридер для каждого источника. В начале каждого аудиосегмента случайно выбирается один источник. Внутри сегмента кадры читаются последовательно из выбранного источника.

Это означает:
- Разные сегменты могут быть из совершенно разных видеофайлов — без ручного монтажа
- Детекция сцен (при включении) выполняется по всем загруженным файлам
- Datamosh-предобработка использует первый файл из выбора как источник P-кадров
- Файлы с разным разрешением и частотой кадров одинаково валидны; каждый читается в своём нативном темпе и масштабируется до разрешения вывода

Лейбл в сайдбаре показывает имя файла при одном файле и счётчик (`3 files loaded`) при нескольких.

---

### 3.15 Обработка тишины

Управляет тем, как движок обрабатывает кадры во время сегментов **SILENCE** длиннее 1 секунды.

Расположено в секции **Cut Logic**, под *Silence Treatment*.

| Вариант | Поведение |
|---|---|
| **Dim (darken)** | Умножает все значения пикселей на 0.6, снижая яркость. Исходное поведение. Хорошо для поддержания визуального контраста между активными и тихими моментами. |
| **Soft Blur** | Применяет гауссово размытие (ядро 15×15). Изображение остаётся полноярким, но теряет резкость — создаёт мечтательный мягкий фокус в тихих фрагментах. |
| **Blur + Dim** | Более лёгкое размытие (11×11) в сочетании с коэффициентом яркости 0.7. Самая сильная обработка — расфокус и затемнение одновременно. |
| **None** | Без обработки. Тихие сегменты рендерятся идентично активным. Используйте, когда тишина композиционно намеренна. |

## 4. Disc VPC 01 R-T — Реалтайм

### 4.1 Интерфейс

Окно разделено на две области:

**Левая панель (258 px, фиксированная):**
- Секция Setup — выбор аудиоустройства, загрузка видео и оверлеев
- Секция Controls — старт/стоп, слайдеры chaos и sensitivity
- Секция Effects — переключатели эффектов, частота оверлеев, последовательный режим

**Правая панель (расширяемая):**
- Живой видеовывод — масштабируется с окном
- Полоса индикаторов — beat, RMS, gate, FPS, спектральные полосы
- Консоль — лог с временными метками

### 4.2 Настройка

**Выбор аудиоустройства**

Выпадающий список показывает все аудиовходы, обнаруженные системой:
- Микрофоны
- Линейные входы
- WASAPI loopback-устройства (например, «Стерео микшер», виртуальные кабели, VoiceMeeter) для захвата системного аудио

Выберите нужное устройство до нажатия START. Кнопка **↺** обновляет список устройств.

**Смена устройства во время работы:** Выберите новое устройство из выпадающего списка пока движок активен. Аудиопоток переключается немедленно и автоматически начинается рекалибровка.

**📁 Video** — загружает один или несколько видеофайлов для воспроизведения и обработки. Поддерживаемые форматы: `.mp4`, `.avi`, `.mov`, `.mkv`, `.flv`, `.wmv`, `.mpg`, `.mpeg`. Удерживайте `Ctrl` или `Shift` для выбора нескольких файлов.

При загрузке нескольких файлов движок держит отдельный последовательный ридер для каждого. Случайные склейки перепрыгивают к случайному кадру из случайно выбранного источника. Последовательное воспроизведение остаётся в текущем источнике до его конца. Предкеш запускается по всем источникам параллельно при старте. Лейбл показывает `N videos loaded` при более чем одном файле.

**📁 Overlays** — загружает папку с изображениями оверлеев (те же форматы, что в §3.9).

---

### 4.3 Аудиокалибровка

При запуске движок входит в **период калибровки** длительностью около 2 секунд (86 аудиочанков при 44100 Гц). В это время:

- Движок измеряет уровень фонового шума выбранного аудиоустройства
- Все склейки и эффекты подавляются
- Индикатор gate показывает `CALIBRATING (N)...` с обратным отсчётом

После завершения калибровки движок устанавливает порог гейта равным `4 × 95-й процентиль RMS`, измеренного в процессе. Это гарантирует, что движок игнорирует фоновый шум и реагирует только на реальный аудиосигнал.

Кнопка **Recalibrate** сбрасывает состояние калибровки и начинает новый цикл измерений. Используйте при смене шумовой обстановки или ручной смене аудиоустройства.

---

### 4.4 Управление и параметры

**▶ START / ⏹ STOP**

START начинает захват аудио и обработку видео. Видеофайл должен быть загружен заранее. STOP останавливает всё. Видеоисточник остаётся загруженным после остановки — нажмите START снова для возобновления без перезагрузки файлов.

---

**Chaos** *(0.0 – 1.0)*

Масштабирует вероятность срабатывания всех эффектов и увеличивает частоту случайных видеосклеек. При `0.0` движок реагирует минимально; при `1.0` — максимально на каждое аудиособытие.

---

**Sensitivity** *(0.2 – 5.0)*

Множитель, применяемый к автокалиброванному порогу гейта:

```
effective_gate = calibrated_gate × sensitivity
```

При `1.0` гейт находится на откалиброванном значении. Увеличьте sensitivity (большой множитель) чтобы требовать более громкого аудио для реакции — полезно в шумной обстановке. Уменьшите (к `0.2`) чтобы реагировать на более тихие сигналы.

---

**Sequential** (переключатель)

При **активном** состоянии (кнопка нажата/подсвечена): движок воспроизводит видео последовательно, кадр за кадром, без случайных склеек. Эффекты по-прежнему применяются.

При **неактивном** (по умолчанию): движок воспроизводит последовательно, но делает склейку к случайному кадру при аудиособытиях (IMPACT, BUILD, DROP или SUSTAIN с битом).

---

### 4.5 Эффекты

Каждый эффект — кнопка-переключатель. Активные эффекты отображаются подсвеченными (тёмный фон, белый текст). Неактивные — в стандартном поднятом состоянии.

| Эффект | Описание |
|---|---|
| **RGB Shift** | Смещает красный и синий каналы горизонтально, пропорционально интенсивности звука |
| **Flash** | Кратковременно переводит кадр к белому или чёрному на транзиентных пиках |
| **Stutter** | На битовых событиях смешивает текущий кадр с недавним буферизованным кадром — кратковременный визуальный статтер |
| **Pixel Sort** | Сортирует пиксели в горизонтальных полосах по яркости |
| **Datamosh** | Использует оптический поток для деформации движения предыдущего кадра на текущий |
| **Overlays** | Накладывает случайные изображения из загруженной папки оверлеев |

**Ovl Freq** *(0.0 – 1.0)* — вероятность размещения оверлея на каждом кадре при активном аудио и включённом эффекте Overlays.

**Chroma Key** — удаление фона из изображений оверлеев. Использует те же режимы и параметры, что и §3.9: `none`, `dominant`, `secondary`, `manual`. Слайдеры Tolerance, Edge Softness и Manual Key Color доступны в секции Chroma Key панели эффектов.

---

### 4.6 Индикаторы

Полоса индикаторов под видеовыводом отображает состояние аудиоанализа в реальном времени.

**● BEAT** — кратко мигает красным при каждом обнаруженном бит-событии.

**RMS: 0.0000** — текущий сглаженный уровень RMS аудиосигнала (экспоненциальное взвешивание).

**GATE / CALIBRATING** — после калибровки показывает значение действующего порога гейта. Становится зелёным, когда RMS превышает гейт (аудио активно). Во время калибровки показывает обратный отсчёт.

**FPS: --** — кадров в секунду цикла обработки видео, обновляется каждую секунду.

**Спектральные полосы:**

| Полоса | Цвет | Частотный диапазон |
|---|---|---|
| **B** (Бас) | Синий | 20 – 300 Гц |
| **M** (Средние) | Зелёный | 300 – 3000 Гц |
| **T** (Высокие) | Красный | 3000 – 16000 Гц |
| **F** (Плоскостность) | Янтарный | Спектральная плоскостность (0 = тональный, 1 = шум) |

Все полосы нормализованы к максимальному значению, наблюдавшемуся за сессию. Полосы отображаются только при уровне аудио выше гейта.

---

### 4.7 Полноэкранный режим и второй монитор

**Полноэкранный режим (F11 или кнопка ⛶)**

Открывает отдельное окно на весь экран. Видео масштабируется заполняя экран. Движок должен быть запущен. Нажмите **ESC** или кнопку на экране для выхода.

**Второй монитор (F12)**

Открывает второе выходное окно, расположенное правее основного дисплея (x=1920 при ширине экрана > 1920). Видео независимо масштабируется под размеры этого окна. Изменяйте размер или перемещайте окно вручную под нужный монитор. Закройте повторным нажатием F12 или остановкой движка.

Оба окна обновляются в каждом кадре синхронно с главным окном.

---

### 4.8 Горячие клавиши

| Клавиша | Действие |
|---|---|
| `F11` | Переключить полноэкранный вывод |
| `ESC` | Выйти из полноэкранного режима |
| `F12` | Переключить вывод на второй монитор |

## 5. Справочник типов сегментов

Оба инструмента используют один классификатор сегментов. Понимание типов сегментов помогает предсказать, какие эффекты сработают.

| Тип | Условие | Типичный момент в аудио |
|---|---|---|
| **IMPACT** | Короткая длительность (< 0.3 с) + громкий + резкая атака | Удар барабана, снейр, резкий транзиент |
| **BUILD** | Нарастающий тренд RMS за последнее время | Нарастание перед дропом, крещендо |
| **DROP** | Падающий RMS после громкого периода | Спад после пика, брейкдаун |
| **SUSTAIN** | Громкий (выше порога), без сильного тренда | Длинная нота, аккорд, стабильный бит |
| **NOISE** | Высокая спектральная плоскостность (> 1.5 × среднее) | Дисторшн, белый шум, плотная текстура |
| **SILENCE** | RMS ниже порога | Тихий фрагмент, пауза между фразами |

Эффекты объявляют, на какие типы сегментов они реагируют. Эффект не сработает на типе сегмента, не входящем в его список триггеров, вне зависимости от настройки chance.

## 6. Рекомендуемые рабочие процессы

### Disc VPC 01 — Итеративное редактирование

1. Загрузите аудио и видеоисточник(и)
2. Выберите встроенный пресет, соответствующий задуманному стилю
3. Нажмите **DRAFT** для быстрого 5-секундного превью в 480p
4. Оцените в мониторе. Скорректируйте Beat Threshold если склейки слишком частые или редкие
5. Включайте/выключайте конкретные эффекты и регулируйте их слайдеры chance
6. Нажмите **PREVIEW** для проверки в полном качестве
7. Повторяйте шаги 4–6 до достижения нужного результата
8. Нажмите **RENDER FULL VIDEO**

### Disc VPC 01 R-T — Живое выступление

1. Загрузите видеофайл(ы) (в идеале — абстрактный, зацикливаемый или богатый клипами материал)
2. По желанию загрузите папку с оверлеями
3. Выберите подходящее аудиоустройство (используйте system loopback для реакции на DAW/плеер)
4. Нажмите **▶ START** и дождитесь завершения калибровки (~2 секунды)
5. Отрегулируйте **Sensitivity** если движок реагирует на тишину или не реагирует на аудио
6. Включите эффекты и настройте **Chaos** по вкусу
7. Используйте **F11** для полноэкранного вывода или **F12** для второго монитора
8. Нажимайте **Recalibrate** при значительных изменениях окружающей обстановки или смене устройства

## 7. Устранение неполадок

**Движок реагирует на тишину / срабатывает постоянно**

Шумовой гейт не откалибровался должным образом. Нажмите **Recalibrate** и подождите 2 секунды. Если проблема сохраняется, увеличьте **Sensitivity** (больший множитель повышает гейт).

**Чёрный экран при повторной загрузке видео**

Загрузите новый видеофайл через кнопку **📁 Video**. Предыдущий источник закрывается автоматически.

**Аудиоустройство не отображается в списке**

Нажмите кнопку **↺** для обновления. В Windows убедитесь, что устройство включено в настройках звука. Для системного loopback включите «Стерео микшер» в Панели управления Windows или используйте виртуальный кабель.

**`Illegal Audio-MPEG-Header` в консоли**

Безобидное предупреждение FFmpeg о некорректных заголовках MP3. Аудио декодируется корректно. Предупреждение можно игнорировать.

**Медленный рендер / низкий FPS в R-T**

Отключите вычислительно затратные эффекты (Datamosh, Pixel Sort). Уменьшите разрешение исходного видео. Закройте другие GPU-интенсивные приложения.

**`madmom` не устанавливается на Python 3.11+**

Используйте Python 3.10.x точно. `madmom` требует Cython-расширений, которые не собираются на более новых версиях Python.

**Эффекты никогда не срабатывают**

Убедитесь, что хотя бы один эффект включён, движок завершил калибровку, а аудиосигнал выше гейта (индикатор RMS становится зелёным). Увеличьте Chaos или уменьшите Sensitivity при необходимости.
